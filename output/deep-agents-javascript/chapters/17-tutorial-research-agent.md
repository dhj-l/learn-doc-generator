## 💡 核心概念

### 17.1 项目概述

**用一个类比来理解：**

> 想象你是一名记者，需要撰写一篇深度报道。你会先做大量的资料搜集——浏览网页、阅读论文、查找统计数据。然后你会整理这些素材，提炼出关键观点。最后你将这些观点组织成一篇有结构的文章。这个流程——搜索、整理、生成——就是深度研究助手的工作流程。
>
> 本章的实战项目将构建一个深度研究助手（Deep Research Agent），它能够接收一个研究主题，自动执行多轮搜索、整理搜索结果、生成结构化报告。

### 17.2 项目架构

这个研究助手由三个核心组件组成，按照数据处理流程依次排列：

1. **搜索工具（web_search）** —— 执行网络搜索，获取与研究主题相关的信息。它是整个流程的起点，决定了研究报告的质量基础。
2. **摘要工具（summarize）** —— 对搜索结果进行提炼，去除冗余信息，提取关键观点和数据。
3. **报告生成工具（generate_report）** —— 将整理后的信息按照特定的结构组织成最终的研究报告。

**项目架构说明：** 在实际使用中，研究助手会根据用户的问题自动触发搜索流程。它先使用不同的关键词进行多轮搜索以获取全面的信息，然后对搜索结果进行摘要和整理，最后将整理后的信息按照报告格式组织成最终的研究报告。整个过程不需要用户干预，Agent 的 LLM 自动判断何时搜索、何时整理、何时生成报告。

```
用户输入研究主题
    │
    ▼
┌──────────────────┐
│  搜索阶段         │  ← 执行多次搜索，覆盖不同角度
│  调用搜索工具 N 次  │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│  整理阶段         │  ← 对搜索结果进行摘要和分类
│  调用摘要工具     │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│  生成阶段         │  ← 将整理后的信息组织为报告
│  调用报告工具     │
└──────┬───────────┘
       │
       ▼
   返回最终报告
```

### 17.3 核心代码实现

```typescript
import { createDeepAgent } from "deepagents";
import { tool } from "langchain";
import { z } from "zod";

// ===== 工具定义 =====

// 搜索工具 —— 模拟网络搜索，获取与研究主题相关的信息
// 在实际生产环境中，这里应该接入真实的搜索 API（如 SerpAPI、Bing Search 等）
// 每次搜索使用不同的关键词组合，从不同角度获取信息
const searchTool = tool(
  async ({ query }: { query: string }) => {
    // 在实际应用中，这里会调用真实的搜索 API
    // 此处简化为返回模拟结果，方便你快速测试
    return `搜索结果：\n1. ${query} 的相关资料一\n2. ${query} 的相关资料二\n3. ${query} 的相关资料三`;
  },
  {
    name: "web_search",
    description: "搜索网络获取与研究主题相关的信息，每次搜索使用不同关键词",
    schema: z.object({
      query: z.string().describe("搜索关键词，应该具体且有针对性，如 'AI 医疗 2025 进展'"),
    }),
  }
);

// 摘要工具 —— 对文本内容进行提炼
// 将搜索结果中的关键信息提取出来，去掉冗余内容
// 摘要的质量直接影响最终报告的质量
const summarizeTool = tool(
  async ({ text }: { text: string }) => {
    // 模拟摘要生成逻辑
    // 实际应用中应该调用 LLM 进行摘要
    return `摘要：对"${text.slice(0, 50)}..."进行分析后，提炼出以下关键信息：\n- 要点一\n- 要点二\n- 要点三`;
  },
  {
    name: "summarize",
    description: "对研究材料进行摘要提炼，提取关键信息，去除冗余内容",
    schema: z.object({
      text: z.string().describe("需要摘要的文本内容"),
    }),
  }
);

// 报告生成工具 —— 将整理后的信息组织成报告
const reportTool = tool(
  async ({ title, content }: { title: string; content: string }) => {
    return `# ${title}\n\n## 概述\n${content}\n\n## 结论\n基于以上研究，可以得出以下结论...`;
  },
  {
    name: "generate_report",
    description: "根据整理好的信息生成结构化研究报告",
    schema: z.object({
      title: z.string().describe("报告标题"),
      content: z.string().describe("报告正文内容"),
    }),
  }
);

// ===== 创建 Agent =====

const agent = createDeepAgent({
  model: "anthropic:claude-sonnet-4-6",
  tools: [searchTool, summarizeTool, reportTool],
  systemPrompt: `你是一个深度研究助手。

## 工作流程
1. **搜索阶段**：当用户给出研究主题后，先进行多次搜索，覆盖不同角度。每次搜索使用不同的关键词，确保全面覆盖
2. **整理阶段**：对搜索结果进行摘要和分类整理，去除重复和低质量的信息
3. **生成阶段**：将整理后的信息组织成结构化报告，包括引言、正文分析和结论

## 行为规则
- 每次搜索前，先思考还有哪些角度需要覆盖，不要满足于一次搜索
- 搜索完成后，对收集到的信息进行摘要，提取关键观点和数据
- 最终输出格式化的研究报告，包含标题、概述、分析和结论
- 如果信息不足以支撑报告，继续搜索直到信息充足
});

// ===== 运行 =====

async function main() {
  // 用户提出研究主题
  // Agent 会自动执行搜索→整理→生成的三阶段流程
  const result = await agent.invoke({
    messages: [
      { role: "user", content: "研究人工智能在医疗领域的最新应用进展" },
    ],
  });

  // 输出 Agent 的完整响应，包括中间的工具调用结果和最终生成的研究报告
  // Agent 会自动执行多次搜索→摘要整理→报告生成的完整流程
  // 最终输出的完整报告包含了引言、正文分析和结论等完整的结构
  // 让用户能够快速了解研究的主要发现和结论
  for (const msg of result.messages ?? []) {
    if (msg.content && typeof msg.content === "string") {
      console.log(`[${msg._getType().toUpperCase()}]\n${msg.content}\n`);
    }
  }
}

main().catch(console.error);
```

## 🚀 项目应用场景

深度研究助手可以应用于多种实际场景：

- **市场调研**：快速收集竞争对手信息、行业趋势和用户反馈，生成市场分析报告
- **学术研究**：帮助研究人员快速了解某个领域的研究现状、关键论文和研究方向
- **技术评估**：在采用新技术前，自动搜索技术文档、社区讨论和最佳实践
- **投资分析**：收集公司信息、财务数据和行业新闻，辅助投资决策

## 💡 使用提示

为了让研究助手发挥最佳效果，建议注意以下几点：

1. **主题要具体**："人工智能在医疗领域的应用"比"人工智能"更容易得到有深度的报告
2. **引导多角度搜索**：在提问时可以提示 Agent 从不同角度搜索，如"请从技术、市场和伦理三个角度研究这个主题"
3. **迭代优化**：如果第一版报告不够深入，可以要求 Agent 继续补充搜索特定方面

## ⚡ 进阶技巧

### 技巧一：并行搜索

为了提高搜索效率，可以让 Agent 同时发起多个搜索请求，而不是逐个执行。在系统提示中引导 Agent 同时搜索多个关键词。

### 技巧二：带缓存的搜索

如果多次搜索相同的关键词，结果应该是一样的。可以在搜索工具中添加缓存逻辑，避免重复请求。

**Q1：深度研究助手的工作流程分为哪三个阶段？**
> A：搜索阶段（执行多次搜索覆盖不同角度）、整理阶段（对搜索结果进行摘要和分类）、生成阶段（将信息组织成结构化报告）。

**Q2：为什么需要多次搜索而不是一次搜索？**
> A：一次搜索只能覆盖一个角度。通过多次搜索不同关键词，可以从多个维度全面了解研究主题，避免遗漏重要信息。

**Q3：摘要工具在流程中起什么作用？**
> A：将原始的搜索结果进行提炼，提取关键信息，去除冗余内容，为后续的报告生成提供高质量的材料。

## 🐛 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| Agent 只搜索一次就生成报告 | 系统提示中没有强调多次搜索的必要性 | 在系统提示中明确要求"至少搜索 3 次，覆盖不同角度" |
| 报告内容过于简略 | 搜索到的信息不足 | 增加搜索次数，使用更具体的关键词 |
| 工具调用顺序混乱 | 未在系统提示中定义清晰的流程 | 在系统提示中明确定义工作的阶段和顺序 |

## 🔍 调试与排查指南

如果研究助手的行为不符合预期，可以从以下几个方面排查：

1. **工具未按顺序调用**：检查系统提示中的流程描述是否清晰，确保每个阶段的触发条件写清楚了
2. **搜索结果不够全面**：检查搜索工具返回的数据量是否足够，必要时增加搜索关键词的多样性
3. **报告内容不完整**：检查报告工具的结构定义是否覆盖了所有必要的章节
4. **Agent 过早生成报告**：在系统提示中明确要求"至少执行 N 次搜索后再生成报告"，或者使用 write_todos 工具将搜索步骤列为待办事项，确保每一步都完成
