## 💡 核心概念

### 19.1 项目概述

**用一个类比来理解：**

> 想象你是一位数据分析师。你的日常工作流程是：先获取原始数据（从 CSV、Excel 或数据库中），然后清洗数据（处理缺失值、去除异常数据），接着进行分析（计算统计指标、找出趋势和规律），最后将结果通过图表可视化并撰写分析报告。
>
> 本章的实战项目将构建一个数据分析 Agent，它可以自动完成从数据加载到生成最终分析报告的整个工作流程和各个步骤。你只需要告诉它数据在哪里、想要分析什么，它就会自动完成所有分析步骤，大大降低了数据分析的门槛和所需的时间成本，让非技术人员也能轻松进行数据分析工作。

**核心能力说明：**

数据分析 Agent 的三个核心工具分别对应数据分析流程中的不同关键步骤和主要工作阶段内容以及相应要求：

1. **load_data（数据加载工具）** —— 从用户指定的文件路径或数据源加载数据集，返回数据的基本结构信息，包括数据的总行数、列数、各列的名称和对应的数据类型，为后续的分析步骤做好准备。
2. **analyze_data（数据分析工具）** —— 对用户指定的一个或多个数据列执行统计分析计算，返回平均值、中位数、标准差、最大值和最小值等关键统计指标。平均值反映数据的集中趋势，标准差反映数据的离散程度，这些指标帮助用户快速了解数据的分布特征。
3. **visualize（数据可视化工具）** —— 根据分析结果和数据特征自动生成合适的图表描述，支持四种图表类型。柱状图（bar）适合不同类别之间的数值对比，折线图（line）适合展示数据随时间的变化趋势，饼图（pie）适合展示各部分在整体中的占比分布，散点图（scatter）适合探索两个变量之间的相关性关系。

这三个工具配合使用，可以完成从原始数据到分析报告的完整流程。Agent 会按照"加载→分析→可视化"的顺序依次调用它们，最终输出完整的分析报告和图表建议。

### 19.3 系统提示设计

数据分析 Agent 的系统提示定义了清晰的工作流程和注意事项。工作流程要求 Agent 按加载→探索→分析→可视化→报告的顺序执行，避免跳过关键步骤。注意事项中强调了在分析前必须先了解数据结构，以及选择合适的图表类型。这些指导规则显著提升了 Agent 的分析质量和结果的可信度。

### 19.2 核心代码实现

```typescript
import { createDeepAgent } from "deepagents";
import { tool } from "langchain";
import { z } from "zod";

// 数据加载工具 —— 从指定数据源加载数据集并返回结构信息
// 返回内容包括数据的总行数、列数、各列的名称和对应的数据类型
// 在实际生产环境中，这里应该连接真实的数据库或读取本地的 CSV 文件
const loadData = tool(
  async ({ source }: { source: string }) => {
    return `已加载数据集：${source}\n数据量：1000 行，8 列\n列名：id, date, value, category, region, sales, profit, rating`;
  },
  {
    name: "load_data",
    description: "从指定来源加载数据集，返回数据的行数、列名和类型等结构信息",
    schema: z.object({
      source: z.string().describe("数据的来源位置，如文件名、URL 或数据库连接串"),
    }),
  }
);

// 数据分析工具 —— 对用户指定的数据列执行统计分析计算并返回结果
// 返回的关键统计指标包括：平均值、中位数、标准差、最大值和最小值
// 平均值反映数据的集中趋势，标准差反映数据的离散程度大小
// 最大值和最小值帮助用户快速了解数据的整体分布范围
// 这些指标可以帮助用户快速了解数据的分布特征和集中趋势
const analyzeData = tool(
  async ({ columns, operation }: { columns: string; operation: string }) => {
    return `分析结果：\n- 对 ${columns} 执行 ${operation} 分析\n- 平均值：125.6\n- 中位数：98.3\n- 标准差：45.2\n- 最大值：356.0\n- 最小值：12.1`;
  },
  {
    name: "analyze_data",
    description: "对指定列执行统计分析，返回关键统计指标",
    schema: z.object({
      columns: z.string().describe("要分析的列名，逗号分隔"),
      operation: z.string().describe("分析操作，如 sum、avg、count、stats"),
    }),
  }
);

// 可视化工具 —— 根据数据的特征自动生成合适的图表描述
// 自动选择最合适的图表类型来展示不同数据之间的关系和趋势
// 柱状图适合数值对比，折线图适合趋势展示，饼图适合占比分析，散点图适合相关性探索
const visualize = tool(
  async ({ chartType, columns }: { chartType: string; columns: string }) => {
    return `图表建议：使用${chartType}图展示 ${columns} 的关系。\nX轴：${columns.split(",")[0]}\nY轴：${columns.split(",")[1] || columns.split(",")[0]}\n标题：${columns} 分析`;
  },
  {
    name: "visualize",
    description: "生成数据可视化图表的描述，支持多种图表类型",
    schema: z.object({
      chartType: z.enum(["bar", "line", "pie", "scatter"]).describe("图表类型"),
      columns: z.string().describe("用于图表的列名"),
    }),
  }
);

const agent = createDeepAgent({
  model: "anthropic:claude-sonnet-4-6",
  tools: [loadData, analyzeData, visualize],
  systemPrompt: `你是一个数据分析助手。你可以帮助用户加载、分析、可视化和报告数据。

## 工作流程
1. 加载数据 —— 先加载用户指定的数据集，了解数据的基本结构
2. 探索数据 —— 了解数据的列名、数据类型和基本统计信息
3. 分析数据 —— 根据用户的需求执行统计分析和计算
4. 可视化 —— 选择合适的图表类型生成可视化描述
5. 输出分析报告 —— 将分析结果组织成完整的报告

## 注意事项
- 在分析之前先了解数据的结构，不要盲目分析
- 选择图表时考虑数据的类型和要表达的关系
- 分析报告应该包含数据概况、分析方法和结论`,
});

async function main() {
  // 用户提出数据分析需求，Agent 会自动执行加载、分析和可视化流程
  const result = await agent.invoke({
    messages: [
      { role: "user", content: "分析销售数据中的趋势，并按地区生成柱状图" },
    ],
  });
  // 输出最终的数据分析结果和图表建议
  // Agent 会自动完成加载数据、探索结构、执行分析和生成可视化等完整流程
  // 用户只需要告诉 Agent 数据在哪里以及想要分析什么问题即可
  // 适合快速探索性数据分析、定期报告生成和运营监控等场景
  console.log(result.messages.at(-1)?.content);
}

main().catch(console.error);
```

### 19.3 代码说明

上述代码实现了数据分析 Agent 的三个核心工具。load_data 负责获取数据并返回结构信息，analyze_data 执行统计计算返回关键指标，visualize 根据数据特征生成合适的图表描述。Agent 会按照数据加载、探索、分析、可视化的顺序依次调用这些工具，最终输出完整的分析报告。

## 🚀 项目应用场景

数据分析 Agent 可以应用于多种实际场景：

- **销售分析**：自动加载销售数据，分析各地区的销售趋势和业绩排名
- **财务报告**：分析财务数据，计算各项财务指标，生成月度或季度财务报告
- **用户行为分析**：分析用户使用数据，找出用户行为模式和产品改进方向
- **运营监控**：定时分析运营数据，自动生成运营日报和周报，辅助管理决策
- **数据质量检查**：自动检测数据中的异常值、缺失值和重复记录，生成数据质量报告

## 💡 使用提示

为了让数据分析 Agent 发挥最佳效果，建议注意以下几点：

1. **明确分析目标**：告诉 Agent 你想要回答什么问题（如"哪个地区的销售额最高"），而不是笼统地说"分析数据"
2. **指定图表类型**：如果你有偏好的图表类型，可以在请求中指定（如"用柱状图展示各地区的销售额对比"）
3. **检查数据质量**：在分析前先让 Agent 检查数据中是否有缺失值或异常值

## ⚡ 进阶技巧

### 技巧一：数据采样分析

对于大数据集，先进行采样分析可以快速了解数据分布，再决定是否需要全量分析。这可以节省大量的处理时间和计算资源，特别是在处理上百万行数据时效果尤为明显。

### 技巧二：异常值检测

在分析工具中添加异常值检测逻辑，自动标记并报告数据中的异常值。异常值可能由数据录入错误、传感器故障或特殊事件引起，如果不加以处理，会对分析结果的准确性和可靠性产生显著影响。

**Q1：数据分析 Agent 的典型工作流程是什么？**
> A：加载数据 → 探索数据结构 → 执行分析 → 可视化 → 生成分析报告，共五个步骤依次执行。

**Q2：为什么需要先探索数据再分析？**
> A：了解数据的结构（列名、数据类型、缺失值情况）有助于选择正确的分析方法，避免因数据质量问题导致错误的分析结论。

**Q3：可视化工具在流程中起什么作用？**
> A：将抽象的数据分析结果转化为直观的图表，帮助用户更容易理解数据中的趋势、模式和异常。

## 🐛 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 数据加载失败 | 指定的数据源路径不正确或格式不支持 | 检查数据源路径，确保格式受支持 |
| 分析结果与预期不符 | 未正确理解数据结构和业务含义 | 先执行数据探索步骤，了解数据含义后再分析 |
| 图表类型选择不当 | 未根据数据类型选择合适的图表 | 在可视化工具参数中加入 chartType 枚举 |
| 分析结果偏差大 | 数据中存在异常值未被及时处理 | 在分析前先执行数据清洗和异常值检测步骤 |
| Agent 跳过探索步骤 | 系统提示中未强调探索的必要性 | 明确要求先探索数据结构再进行分析 |

## 🔍 调试与排查指南

如果数据分析 Agent 的行为不符合预期，可以从以下几个方面排查：

1. **数据加载失败**：检查 load_data 工具中的模拟数据逻辑是否正确，或者在实际部署时检查数据源的连接配置
2. **分析结果异常**：检查 analyze_data 工具的统计计算逻辑是否正确，确认数据中的异常值没有被忽略
3. **图表选择不当**：在 visualize 工具的 schema 中限制 chartType 参数的可选值，引导 Agent 选择更合适的图表类型
4. **Agent 跳过数据探索步骤**：在系统提示中强调"在分析之前必须先加载和探索数据"，确保流程完整性
