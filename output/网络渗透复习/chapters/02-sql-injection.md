# SQL 注入大题

基于文档中两套 SQL 注入实训题目整理。

---

## 实训一

**环境**：DVWA 漏洞演练靶场，Low 等级，SQL Injection 模块。功能：输入 User ID 查询对应人员姓名。

**后端 SQL 语句**：

```sql
SELECT first_name, last_name FROM users WHERE user_id = '$id';
```

### 题目 1

测试人员在 ID 输入框提交单引号载荷 `'`，页面返回数据库语法报错。解释报错原理，并说明该现象证明了何种漏洞存在。

**答案**：
- **原因**：输入单引号 `'` 会闭合后台 SQL 语句中原有的单引号，破坏原有 SQL 语法结构，数据库执行非法语法抛出报错
- **结论**：页面出现数据库报错，证明页面未过滤用户输入，存在**字符型 SQL 注入漏洞**

> 可以这样理解：原始 SQL 是 `WHERE user_id = '你的输入'`，你输入一个 `'` 后变成了 `WHERE user_id = '''`——多了一个引号，语法乱了，数据库就报错了。

### 题目 2

攻击者依次输入两条载荷，对应返回不同结果，写出两条载荷的含义和执行目的。

**① `' order by 3 --`**
- 含义：利用 `order by` 判断数据表字段数量，数字 3 代表查询数据表是否存在第 3 列
- 执行目的：确定联合查询时可展示数据的字段数，为 `union` 查询做准备

**② `' union select database(),version() --`**
- 含义：联合查询拼接两条查询语句
- 执行目的：`database()` 获取当前网站数据库名称，`version()` 获取后端 MySQL 数据库版本

> 可以把 `order by` 想象成"试座位"——试第 1 排有座（不报错），试第 2 排也有座，试第 3 排没座了（报错），就知道一共 2 排（2 个字段）。
> `union` 就像"拼桌"——在原有查询结果后面拼上你想要的查询结果。

### 题目 3

管理员需要修复该 SQL 注入漏洞，给出四项整改方案，指出唯一能彻底根治的方案，并说明其余三项仅能缓解的原因。

- A. 后端代码对单引号、注释符、SQL 关键字进行转义过滤
- B. 重构数据库交互代码，采用**预编译参数化查询**执行 SQL
- C. 前端部署 WAF，拦截 union、select 等注入特征词
- D. 收缩数据库账号权限，仅开放查询权限

**根治方案：B**

**各方案分析**：
- **A**：输入字符转义仅表层过滤特殊符号，存在编码、变形绕过手段，只能临时降低攻击成功率
- **B ✅**：参数化查询将用户输入与 SQL 语法完全隔离，恶意字符无法篡改语句逻辑，从底层根除注入漏洞
- **C**：WAF 关键词拦截可通过大小写替换、URL 编码等方式绕过，仅起到基础防护效果
- **D**：数据库最小权限只能限制漏洞造成的破坏范围，无法阻止注入漏洞本身被利用

> 预编译为什么能根治？可以这样理解：
> - 普通写法：`"SELECT * FROM users WHERE id = '$id'"`——把用户输入直接拼进 SQL 语句，用户输入什么"文字"，SQL 就执行什么"文字"
> - 预编译：先确定 SQL 的骨架，再把用户输入作为**数据值**传进去——用户输入永远是"值"，永远不会变成"SQL 代码的一部分"

## 实训二

**环境**：DVWA，Low 等级，SQL Injection 模块。后端 SQL 语句同上。

### 题目 1

输入 `'` 报错，输入 `1' and 1=1 #` 正常返回数据，输入 `1' and 1=2 #` 无数据。该漏洞属于数字型还是字符型注入？说明判断依据。

**答案**：属于**字符型注入**。

**依据**：参数用单引号闭合，单引号破坏语法报错；`and 1=1`、`and 1=2` 能区分真假条件，符合字符型注入判定特征。

> 数字型注入不需要引号闭合，参数直接拼接在 SQL 里。这里要加引号闭合，说明是字符型。

### 题目 2

实验中使用语句 `1' order by 2 #`，请回答：
1. `order by` 的作用是什么？
2. `#` 在 SQL 语句中的功能是什么？
3. 该语句执行成功无报错，说明查询表有多少个回显字段？

**答案**：
1. `order by`：猜测数据表字段数量
2. `#`：SQL 注释符，注释后方多余语句避免语法报错
3. 表中有 **2 个**可回显字段

> `#` 的作用是"闭嘴"——把 SQL 语句末尾多余的 `'` 注释掉，不然多一个引号会报错。

### 题目 3

实操语句填空：
1. 查询当前网站数据库名称，需输入完整注入语句：`1' and 1=2 union select ______,2 #`
2. 获取 users 表内账号密码，需要查询字段 user 和 password，写出核心查询函数：`group_concat(________)`

**答案**：
1. `database()`
2. `user,password`

> `group_concat` 的作用是把多行结果合并成一行——好比把一整列的名字写成一串："小明,小红,小刚"，方便一次性显示。

### 题目 4

实操时输入注入语句页面无任何数据回显，排查出两种可能原因并写出解决办法。（任选两条即可）

**① 原因**：union 前后字段数量不一致
**解决**：使用 `order by` 重新确定字段数，匹配两侧列数

**② 原因**：未使用 `and 1=2` 让前半查询失效
**解决**：在 union 前添加 `and 1=2` 清空原有查询结果

**③ 原因**：缺少注释符 `#`，末尾单引号引发语法错误
**解决**：语句结尾添加 `#` 注释多余代码
