# RAG 系统速查表

## 🏗️ RAG 三阶段
```
Indexing → Retrieval → Generation
索引      → 检索       → 生成
```

## 📦 分块策略

| 策略 | chunkSize | 适用 |
|------|-----------|------|
| 固定长度 | 500-1000 | 通用 |
| 递归分块 | 500-1000 | 推荐默认 |
| 语义分块 | 自适应 | 高精度场景 |

## 🔍 检索策略

| 方法 | 优点 | 缺点 |
|------|------|------|
| 语义搜索 | 理解语义 | 可能漏精确匹配 |
| BM25 | 精确匹配 | 不理解同义词 |
| 混合检索 | 互补优势 | 复杂度较高 |
| 重排序 | 精度最高 | 增加延迟和成本 |

## 📊 评估指标

| 指标 | 含义 |
|------|------|
| Faithfulness | 回答是否忠于文档 |
| Relevancy | 回答是否切题 |
| Context Recall | 是否找到相关文档 |
| Context Precision | 相关文档占比 |
