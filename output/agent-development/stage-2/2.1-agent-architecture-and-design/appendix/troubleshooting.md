# Agent 架构常见错误排错指南

---

## 1. Agent 死循环

**现象：** Agent 反复执行相同操作，无法退出
**原因：** 没有设置迭代上限，或工具返回的结果让 Agent 无法做出终止决策
**解决方案：** 设置 `maxIterations` 上限（建议 10-20）；检测重复动作（连续 2 次相同动作 → 强制结束）；添加超时机制

## 2. 工具调用解析失败

**现象：** LLM 输出的 Action 格式不正确，无法解析
**原因：** System Prompt 中格式定义不够精确，或模型 Temperature 太高
**解决方案：** 在 System Prompt 中更精确地定义格式（附带 Few-shot 示例）；降低 Temperature（建议 0-0.3）；使用 Structured Output

## 3. Agent 忘记之前的步骤

**现象：** Agent 在第 5 步忘记了第 2 步的结果，重新执行已完成的步骤
**原因：** 上下文被新的工具结果挤出，或 Observation 过于冗长
**解决方案：** 在每步的 Observation 中包含历史摘要；使用 Structured Output 记录每步状态

## 4. Token 成本爆炸

**现象：** 单次 Agent 任务消耗了大量 Token，远超预期
**原因：** 没有设置 Token 预算，或 Agent 进入了不必要的迭代
**解决方案：** 设置 Token 预算上限（如 100K）；使用更便宜的模型处理简单步骤；压缩中间步骤的 Observation

## 5. Multi-Agent 通信混乱

**现象：** Agent 之间传递的信息不准确，或 A 发给 B 的消息 B 无法理解
**原因：** 消息格式没有标准定义
**解决方案：** 使用 Structured Output 定义消息格式；添加消息校验机制

## 6. ReAct 模式的推理链太浅

**现象：** Agent 只做了 1-2 步推理就给出了答案，忽略了许多需要多步推理的情况
**原因：** System Prompt 没有鼓励深入推理
**解决方案：** 在 Prompt 中加入「请逐步推理」；降低 Temperature；设置最小推理步数

## 7. Plan-and-Execute 的计划过于理想化

**现象：** Planner 生成了完美的计划，但 Executor 在实际执行时发现很多步骤不可行
**原因：** Planner 没有获取环境信息就做规划
**解决方案：** Planner 先进行一轮「环境探测」再制定计划

## 8. Reflexion 中的反思沦为形式

**现象：** 每次反思都说「我应该更仔细」，但下次尝试没有任何实质性改进
**原因：** 反思没有指向具体可操作的改进点
**解决方案：** 强制反思包含具体改进措施：「下一次我应该在 X 步骤之前先做 Y」

## 9. Agent Loop 中混合了不同模式的 Action 格式

**现象：** 同一个 Agent 循环中既有 ReAct 格式又有 Function Calling，导致解析冲突
**原因：** 在 Prompt 中同时定义了两种 Action 格式
**解决方案：** 统一使用一种工具调用模式（优先使用原生 Function Calling）

## 10. 工具调用结果中包含错误没有被正确处理

**现象：** 工具返回了错误信息，Agent 无视错误继续往下执行
**原因：** Agent 把错误信息当作正常结果处理了
**解决方案：** 在工具返回结果中标记 `is_error: true`

## 11. Agent 在复杂任务中过早收敛

**现象：** Agent 完成了任务的一部分就认为任务已完成
**原因：** 任务分解不够细致
**解决方案：** 在 System Prompt 中加入「确认已经完成了任务的所有部分」

## 12. 多个 Agent 同时修改同一状态导致竞态

**现象：** 两个 Agent 同时读取了相同的状态，各自做了修改，后一个覆盖了前一个
**原因：** 缺乏锁机制或状态版本控制
**解决方案：** 使用乐观锁（版本号）；或将状态修改集中到 Supervisor 代理

## 13. Supervisor 代理成为性能瓶颈

**现象：** 所有通信都经过 Supervisor，当 Agent 数量增加时延迟急剧上升
**原因：** Supervisor 模式天然是中心化的
**解决方案：** 使用分层 Supervisor；对于简单通信允许 Agent 直接通信

## 14. Agent 在长对话中丢失了 System Prompt 的影响

**现象：** 对话进行到 20 轮后，Agent 开始忽略 System Prompt 中的指令
**原因：** 长上下文中 System Prompt 的注意力被稀释
**解决方案：** 每隔 N 轮重新注入 System Prompt 的核心指令

## 15. 调试 Agent 时难以复现问题

**现象：** 相同的输入，Agent 有时成功有时失败
**原因：** LLM 的输出有随机性（即使 Temperature=0）
**解决方案：** 设置随机种子；记录完整的 Agent 运行轨迹

## 16. 开发和生产环境的 Agent 行为不一致

**现象：** 本地测试正常，生产环境 Agent 频繁出错
**原因：** 模型版本不同、上下文长度限制不同
**解决方案：** 开发和生产环境使用完全相同的模型和参数配置

## 17. Agent 的决策过于依赖单一信息来源

**现象：** Agent 只根据一次搜索的结果就做出重大决策
**原因：** Prompt 中没有要求 Agent 交叉验证信息
**解决方案：** 在 Prompt 中加入「对于重要决策，请从多个来源验证信息」

## 18. Memory 系统在 Agent Loop 中的使用不当

**现象：** Agent 把所有的中间状态都存入了长期记忆，导致记忆库被污染
**原因：** 没有区分「任务内工作记忆」和「跨会话长期记忆」
**解决方案：** 只在任务完成后将关键信息写入长期记忆；中间状态使用短期工作记忆
