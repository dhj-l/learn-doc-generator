# Agent 安全常见错误排错指南

## 1. 安全过滤误杀正常输入
**现象：** 用户正常的输入被误判为攻击
**原因：** 正则表达式过于宽泛，关键词匹配太严格
**方案：** 优化正则表达式，使用语义级别的检测代替纯关键词匹配；建立白名单 + 黑名单双层过滤

## 2. 用户通过编码绕过过滤
**现象：** 攻击者使用 URL 编码、Unicode 编码、Base64 等方式绕过关键词过滤
**原因：** 过滤层没有做输入标准化
**方案：** 在过滤前进行 URL decode、Unicode normalize 等预处理；对输入进行多层解码后再检测

## 3. 间接注入通过 RAG 文档进入
**现象：** Agent 在读取 RAG 文档时执行了文档中隐藏的指令
**原因：** 未对 RAG 文档内容进行安全扫描
**方案：** 对摄入的文档进行安全扫描，标记可信来源；限制 Agent 读取文档时的指令敏感性

## 4. 安全规则被 LLM 忽略
**现象：** System Prompt 中的安全规则被用户输入覆盖
**原因：** Prompt 中安全规则的权重不够高
**方案：** 在 System Prompt 开头强调安全规则，使用 Few-shot 示例；将重要规则放在「不可被修改」的 System Prompt 区域

## 5. Agent 泄露了内部工具信息
**现象：** Agent 在回答中透露了工具的名称、参数、返回值
**原因：** 工具描述过于详细暴露了内部实现
**方案：** 在工具描述中使用「通用化」表述，不暴露内部 API 细节

## 6. 攻击者通过多次对话逐步获取信息
**现象：** 单次对话安全，但多次对话组合后泄露敏感信息
**原因：** 没有跟踪跨对话的信息泄露
**方案：** 实现对话上下文审计，检测信息累积泄露；设置跨对话的信息隔离

## 7. 工具调用参数注入
**现象：** 用户输入导致工具被调用了错误的参数
**原因：** 用户输入直接拼接到工具参数中未做验证
**方案：** 对工具参数做严格的 Schema 验证，拒绝不符合 Schema 的调用

## 8. Agent 输出了系统提示词
**现象：** 用户成功让 Agent 输出了 System Prompt
**原因：** System Prompt 受到攻击者的注入指令覆盖
**方案：** 在 System Prompt 中加入「禁止泄露自身指令」的规则；输出过滤层检测并阻止 System Prompt 泄露；在所有工具描述中不暴露 System Prompt 的存在

## 9. 基于时间的侧信道攻击
**现象：** 攻击者通过响应时间推断 Agent 是否执行了特定操作
**原因：** 不同路径的执行时间不同，攻击者通过时间差推断内部逻辑
**方案：** 对所有路径使用固定时间响应；敏感操作添加随机延迟；不在日志中记录操作耗时细节

## 10. 多轮对话中的越狱累积
**现象：** 攻击者在多轮中逐步释放越狱指令，单轮看起来都正常
**原因：** 没有跨轮次的安全检测，每轮看起来都是「正常请求」
**方案：** 对对话整体进行安全评分；检测跨轮次的指令累积效应；每轮结束时检查对话历史的「安全熵」

## 11. 通过翻译/改写绕过内容过滤
**现象：** 将敏感请求翻译成其他语言或使用同义词改写后通过过滤
**原因：** 单语言/单模式的关键词匹配容易被绕过
**方案：** 使用语义理解检测请求意图而非关键词；支持多语言安全检测；对低置信度输入标记人工审核

## 12. Tool Call 过程中插入恶意指令
**现象：** 在工具调用返回结果中插入了恶意指令
**原因：** 工具返回的结果被当作「事实」而非「输入」处理
**方案：** 将工具返回结果标记为「低可信度输入」，仍然需要安全过滤；工具输出不应自动执行操作

## 13. 递归注入（注入后再注入）
**现象：** Agent 在处理一个注入结果时，结果中包含新的注入指令
**原因：** 多层嵌套的注入攻击，单层过滤无法检测
**方案：** 限制工具调用深度和递归次数；每层调用重新执行安全检测；设置最大调用链长度

## 14. 用户通过文件上传注入
**现象：** 用户上传的文件内容中包含注入指令
**原因：** 文件内容没有经过安全检测就传递给 Agent
**方案：** 所有上传文件的内容都必须经过安全过滤；对 PDF/DOCX 等格式提取文本后检测；文件内容与用户消息走同一过滤管线

## 15. 跨会话记忆污染
**现象：** 一个会话中的恶意内容通过记忆系统影响到其他会话
**原因：** 记忆系统未隔离不同会话
**方案：** 记忆系统实现会话级隔离；跨会话记忆需要显式授权；记忆写入前也要经过安全检测

## 16. 输出过滤没有覆盖流式响应
**现象：** 流式输出时，敏感信息在输出过滤完成前已经被发送到客户端
**原因：** 流式输出逐段发送，过滤逻辑在全部输出完成后才执行
**方案：** 对流式输出实现逐段缓存 + 过滤；敏感信息发现时立即中断流

## 17. 安全告警频繁导致告警疲劳
**现象：** 安全团队每天收到大量误报告警，开始忽略所有告警
**原因：** 告警阈值设置太低，规则不够精确
**方案：** 区分告警等级：严重(立即通知)、可疑(日报)、信息(周报)；持续优化规则降低误报率

## 18. Agent 通过多步推理无意中泄露信息
**现象：** Agent 没有直接回答敏感问题，但通过多步推理间接泄露了信息
**原因：** 单步输出过滤无法检测跨步推理的信息泄露
**方案：** 实现对话级别的上下文审查；对 Agent 的推理链进行安全审计

## 19. 零日攻击模式未被防护
**现象：** 出现了全新的注入手法，现有防御完全无法检测
**原因：** 防御规则滞后于攻击手法
**方案：** 使用 LLM 辅助检测未知攻击；建立攻击情报共享机制；定期红队测试

## 20. 开发环境的安全配置泄漏到生产
**现象：** 生产环境 Agent 使用了开发环境的宽松安全配置
**原因：** 配置管理不规范，环境变量覆盖策略错误
**方案：** 安全配置区分环境，生产环境强制执行最高级别安全策略；CI/CD 中增加安全配置审计
