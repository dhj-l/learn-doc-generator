# Function Calling 常见错误排错指南

---

## 1. Claude 不调用工具

**现象：** Agent 直接回答了用户的问题，没有调用任何工具，即使问题明显需要工具
**原因：** 工具描述不够清晰，或用户问题不需要工具
**解决方案：**
- 改进工具描述，说明适用场景（「当用户问到天气时使用」比「天气工具」好很多）
- 使用 `tool_choice: { type: 'any' }` 强制调用
- 在 System Prompt 中引导 Claude 使用工具

---

## 2. 调用了错误的工具

**现象：** 拿到了用户问题，但调用了不相关的工具
**原因：** 工具描述有歧义，或多个工具功能重叠
**解决方案：**
- 明确每个工具的边界（「用 A 做 X，用 B 做 Y」）
- 使用 `tool_choice: { type: 'tool', name: 'correct_tool' }` 指定

---

## 3. 工具参数格式错误

**现象：** 工具调用返回 Schema 验证错误
**原因：** LLM 生成了不符合 Schema 的参数（类型不对、缺少必填字段）
**解决方案：**
- 使用 enum 限制参数选项
- 为参数添加清晰的 description（LLM 靠这个理解参数含义）
- 在工具执行前做参数验证：`z.object({...}).parse(args)`

---

## 4. 并行工具调用结果处理错误

**现象：** Agent 第二次调用时无法获得第一次工具调用的结果
**原因：** 没有正确关联 `tool_use_id` 和 `tool_result`
**解决方案：**
- 确保每个 `tool_result` 都有对应的 `tool_use_id`
- 保持 content 数组中 tool_use 和 tool_result 的对应关系

---

## 5. 工具循环不退出

**现象：** Agent 不断调用工具，陷入死循环
**原因：** 工具返回的结果让 Claude 反复调用
**解决方案：**
- 设置最大迭代次数（如 10 次）
- 在工具结果中明确说「操作完成」或「没有更多」
- 使用 `stop_reason === 'end_turn'` 判断正常结束

---

## 6. 流式工具调用中丢失事件

**现象：** 流式响应中工具调用的 `content_block_start` 事件没有触发
**原因：** 流式处理代码遗漏了 `content_block_start` 事件类型
**解决方案：** 确保流式处理代码覆盖所有事件类型；使用 SDK 的 `on('contentBlockStart')` 方法

---

## 7. 工具返回值太大导致 Token 爆炸

**现象：** 工具返回了一个很大的 JSON，后续对话的 Token 消耗急剧增加
**原因：** 工具结果被原样放入了 messages 数组，没有截断或摘要
**解决方案：** 对工具返回值做截断（最大 1000 Token）；提供摘要而非原始数据

---

## 8. 多轮对话中工具上下文混乱

**现象：** 第 3 轮对话时，Agent 把第 1 轮的工具调用结果又用了一次
**原因：** messages 数组中累积了太多 tool_use/tool_result 对，模型分不清哪些是当前的
**解决方案：** 定期清理过期的 tool_use/tool_result 对；使用 `max_tokens` 限制上下文长度

---

## 9. 最终回答没有引用工具结果

**现象：** 工具调用成功了，但 Agent 的最终回答完全忽略了工具返回的数据
**原因：** tool_result 返回后没有给模型足够多的输出 Token 来处理结果
**解决方案：** 增大 `max_tokens`；确保 tool_result 在 messages 中的位置紧跟着用户的提问

---

## 10. 使用 `any` 模式时模型过度调用工具

**现象：** `tool_choice: { type: 'any' }` 模式下，模型连打招呼都调用工具
**原因：** `any` 模式强制模型每次响应都调用工具，包括不需要的场景
**解决方案：** 只在需要结构化输出的轮次使用 `any`，其他轮次用 `auto`；或使用 `tool` 模式指定具体工具

---

## 11. 工具执行超时导致 Agent 卡住

**现象：** Agent 调用工具后长时间没有响应
**原因：** 工具执行时间太长（如 API 调用 30 秒），超过了 Agent 的等待时间
**解决方案：** 设置工具执行超时（如 10 秒）；超时后返回错误结果让 Agent 决定下一步

---

## 12. 工具调用被错误地标记为 `is_error`

**现象：** 工具正常执行了，但 Agent 认为执行失败了
**原因：** `tool_result` 中 `is_error` 参数设置错误
**解决方案：** 只有工具抛出异常时才设置 `is_error: true`；正常返回即使结果为空也不要设为错误

---

## 13. 同一工具在多个步骤中被重复调用

**现象：** Agent 在同一个问题的处理过程中反复调用同一个工具
**原因：** 工具没有缓存机制，或 Agent 没有利用之前的调用结果
**解决方案：** 在工具层面实现结果缓存；在 System Prompt 中提示 Agent「优先使用已有的结果」

---

## 14. 迁移到新模型后工具定义不兼容

**现象：** 之前能用的工具定义在新模型上报 Schema 验证错误
**原因：** 不同模型对工具 Schema 的支持不同（如 Claude 3 vs 4）
**解决方案：** 查阅新模型的工具定义限制；简化 Schema 避免使用高级 JSON Schema 特性

---

## 15. 工具的中文名称导致编码问题

**现象：** 工具名称为中文时调用失败或乱码
**原因：** LLM 对工具名称的编码处理不一致
**解决方案：** 工具名称统一使用英文小写 + 下划线；在 description 中写中文含义

---

## 16. 安全过滤误拦截了合法的工具调用

**现象：** 输入过滤系统将合法的工具参数识别为注入攻击
**原因：** 工具参数中的某些关键词触发了安全规则
**解决方案：** 为系统生成的 tool_call 添加标记，绕过用户输入过滤；只对用户输入做注入检测

---

## 17. 调试时看不到 LLM 选择的工具

**现象：** Agent 行为不符合预期但不知道 LLM 选择了什么工具和参数
**原因：** 没有记录工具的中间状态
**解决方案：** 在开发环境下打印工具调用的完整 JSON；使用 `console.log(JSON.stringify(response, null, 2))`

---

## 18. Function Calling 与其他能力冲突

**现象：** 同时使用工具和结构化输出时，模型行为异常
**原因：** `tool_choice` 和 `response_format` 同时设置时模型不确定优先级
**解决方案：** 避免在同一请求中同时使用工具和结构化输出；需要结构化输出时用工具代替
