# 记忆系统常见错误排错指南

## 1. 对话太长导致 Token 超限
**现象：** Agent 在长对话中报错「context length exceeded」
**原因：** 对话历史不断累积，超出了模型的上下文窗口
**方案：** 使用摘要压缩或滑动窗口策略，保留最近 N 轮对话

## 2. 记忆检索不准确
**现象：** Agent 记错了用户之前提到的信息
**原因：** Embedding 模型质量不足，或相似度阈值设置不当
**方案：** 检查 Embedding 模型质量，调整 topK 和相似度阈值

## 3. 记忆存储了无关信息
**现象：** Agent 记住了大量无用的信息，检索时噪声太多
**原因：** 没有对记忆进行重要性筛选
**方案：** 添加重要性评估，只存储 importance >= 5 的信息

## 4. 跨会话记忆无法访问
**现象：** Agent 在新会话中不记得上次对话的内容
**原因：** 记忆存储在内存中（临时），未持久化到数据库
**方案：** 确保使用持久化存储（向量数据库），而非内存

## 5. 记忆冲突（新旧矛盾）
**现象：** Agent 同时记得「用户喜欢 Python」和「用户喜欢 TypeScript」
**原因：** 没有处理记忆的时间顺序和优先级
**方案：** 存储时添加时间戳，检索时优先最新的记忆

## 6. 工作记忆被过早清空
**现象：** Agent 在任务执行到一半时忘记了中间步骤
**原因：** 工作记忆的 TTL 设置过短
**方案：** 延长工作记忆的 TTL，或在任务完成前不清空

## 7. 记忆检索延迟过高
**现象：** 每次记忆检索花费数秒
**原因：** 向量数据库索引未优化，或数据量过大
**方案：** 添加向量索引（IVF/HNSW），限制检索范围

## 8. 记忆存储成本过高
**现象：** 向量数据库的存储费用不断增长
**原因：** 存储了所有对话记录，没有清理策略
**方案：** 定期清理低价值记忆，使用摘要压缩历史记录

## 9. 无法检索到特定信息
**现象：** 用户问「我之前说过的 XX」，Agent 找不到
**原因：** 语义检索对精确实体名称检索效果不好
**方案：** 结合关键词搜索和语义搜索（混合检索）

## 10. 记忆系统在多用户间混淆
**现象：** Agent 把用户 A 的记忆用到了用户 B 身上
**原因：** 记忆查询未按 user_id 过滤
**方案：** 所有记忆操作必须携带 user_id 参数

## 11. 记忆更新不及时
**现象：** 用户改了偏好，Agent 还是用旧的
**原因：** 记忆更新不是替换而是追加
**方案：** 更新记忆时先删除旧记录再插入新记录

## 12. 摘要压缩丢失关键细节
**现象：** 压缩后 Agent 丢失了重要信息
**原因：** 摘要生成的 prompt 不够详细
**方案：** 分维度压缩（关键事实 + 重要细节 + 时间线）

## 13. 记忆检索的 relevance score 不可靠
**现象：** 高位分数的内容实际不相关
**原因：** Embedding 相似度不等于语义相关性
**方案：** 添加二次重排（reranking）提升质量

## 14. 工作记忆和长期记忆混淆
**现象：** Agent 把临时状态存成了长期记忆
**原因：** 没有明确区分两种记忆的存储逻辑
**方案：** 分别实现工作记忆（临时）和长期记忆（持久化）

## 15. 对话历史中包含工具调用信息
**现象：** 工具调用的详细参数和结果占据了大量 Token
**原因：** 没有对工具调用记录做摘要
**方案：** 压缩工具调用记录为一句摘要：「调用了 X 工具，结果：成功/失败」

## 16. 对话压缩后 Agent 丢失了之前的工具状态
**现象：** Agent 在压缩后的对话中无法继续之前的多步工具操作
**原因：** 压缩摘掉了工具调用的中间状态
**方案：** 在摘要中保留关键的工具调用链：「已搜索 → 找到结果 → 正在分析」

## 17. 用户明确要求忘记某条记忆但系统仍记着
**现象：** 用户说「忘掉我刚才说的 X」，但 Agent 后续仍能回忆起
**原因：** 记忆删除操作没有实际执行向量数据库的 delete
**方案：** 实现「记忆遗忘」API：`memories.delete({ where: { content: { $contains: 'X' } } })`

## 18. 记忆存储未做去重导致重复积累
**现象：** 用户反复说同一件事，记忆系统存了多条相似记录
**原因：** 每次写入都创建新条目，没有检测重复
**方案：** 写入前先检索是否已有相似记忆；存在则更新时间戳而非新建

## 19. 跨语言检索时记忆找回率大幅下降
**现象：** 用户用中文存的记忆，用英文提问时找不到
**原因：** Embedding 模型对跨语言的语义匹配不够好
**方案：** 使用多语言 Embedding 模型（如 BGE-M3）；检索时用同样语言

## 20. 短期记忆和工作记忆的边界不清晰
**现象：** Agent 把工作记忆中的临时变量混入短期记忆，导致上下文爆炸
**原因：** 两种记忆使用了同一个存储空间
**方案：** 明确分离：短期记忆 = conversation history，工作记忆 = 独立 Map 对象
