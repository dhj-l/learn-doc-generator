# RAG 系统常见错误排错指南

## 1. 检索不到相关文档
**现象：** Agent 回答「我找不到相关信息」
**原因：** 分块(chunk)太大或太小，Embedding 模型不匹配
**方案：** 检查分块质量（chunk_size 500-1000），换用更好的 Embedding 模型，尝试混合检索

## 2. 回答出现幻觉（Hallucination）
**现象：** Agent 回答了文档中没有的内容
**原因：** Prompt 约束不够强，或模型过度自信
**方案：** 在 Prompt 中强调「只基于文档回答，如果文档中没有相关信息就说不知道」，使用 Faithfulness 评估

## 3. 回答不够详细
**现象：** Agent 的回答太简短，信息量不足
**原因：** topK 太小或 chunkSize 太小
**方案：** 增加 topK（5→10），增大 chunkSize（500→1000），使用 Multi-hop RAG

## 4. 检索延迟太高
**现象：** 用户等了几秒才看到回答
**原因：** 向量数据库未优化，或网络延迟
**方案：** 使用缓存、优化向量索引（IVFFlat → HNSW）、减少 topK、使用本地向量数据库

## 5. Token 成本太高
**现象：** 每次 RAG 查询消耗大量 Token
**原因：** 检索了太多文档，或文档太长
**方案：** 精简检索结果（减少 topK）、使用 Prompt Caching、用便宜的模型做重排

## 6. 检索到不相关的文档（低精度）
**现象：** topK 结果中有大量不相关的文档
**原因：** Embedding 无法区分细微语义差异
**方案：** 提高相似度阈值、添加关键词预过滤、使用重排序（reranker）

## 7. 混合检索的结果不如纯向量检索
**现象：** 加了关键词搜索后质量反而下降
**原因：** 关键词和语义结果的合并策略不当
**方案：** 使用 Reciprocal Rank Fusion（RRF）合并策略，或使用加权平均

## 8. 查询太短导致检索不准
**现象：** 用户输入「Python」但想找的是「Python 异步编程」
**原因：** 短查询的语义信息不足
**方案：** 使用查询重写（Query Rewrite）扩展用户查询

## 9. 跨语言检索失败
**现象：** 中文查询找不到英文文档中的相关信息
**原因：** Embedding 模型不支持多语言
**方案：** 使用多语言 Embedding 模型（如 multilingual-e5），或先翻译再检索

## 10. PDF/图片中的文字无法检索
**现象：** PDF 中的表格和图片信息无法通过 RAG 检索到
**原因：** 简单文本提取丢失了结构化信息
**方案：** 使用 OCR + 视觉模型提取 PDF 内容，保留文档结构

## 11. 文档更新后索引未同步
**现象：** 文档已经改了但 Agent 还在用旧内容回答
**原因：** 向量数据库未与文档源同步更新
**方案：** 实现文档变更监听（Webhook/轮询），变更后重新索引

## 12. 上下文窗口装不下检索结果
**现象：** 检索出的所有文档放不进 LLM 的上下文
**原因：** topK 太多或文档长度太长
**方案：** 使用上下文压缩（LLM 提取关键信息），或滑动窗口分多次处理

## 13. 重排序（Reranking）增加了不必要的延迟
**现象：** 加入 reranker 后每次查询多花 1-2 秒
**原因：** 对全部检索结果都做了一轮重排
**方案：** 先粗筛（取 topK=20），再重排（取 top=5），只对小量结果重排

## 14. 日志中无法追踪每次检索的来源
**现象：** 不知道 Agent 回答中的某个事实来自哪篇文档
**原因：** 没有记录检索的溯源信息
**方案：** 在回答中添加引用标记（如 [1][2]），日志中记录每次检索的文档 ID

## 15. 文档分块破坏了语义完整性
**现象：** 一个完整的概念被拆到了两个 chunk 中
**原因：** 简单按字符数分块，没有考虑语义边界
**方案：** 使用语义分块（在段落边界处切分），或使用重叠分块（Overlap Chunking）

## 16. 多轮对话中的 RAG 上下文冲突
**现象：** 第二轮的 RAG 结果和第一轮的上下文矛盾
**原因：** 没有清理之前轮次的 RAG 上下文
**方案：** 每轮 RAG 查询清空之前的上下文，只在当前轮次注入检索结果

## 17. 处理流式 RAG 输出时的竞态条件
**现象：** 用户看到的内容先来自 RAG、然后被 LLM 生成覆盖
**原因：** RAG 检索和 LLM 生成没有正确同步
**方案：** 先完成检索再开始流式生成，确保内容顺序正确

## 18. Embedding 维度不匹配
**现象：** 存入向量数据库时维度是 1536，查询时变成了 384
**原因：** 使用了不同的 embedding 模型进行存储和查询
**方案：** 确保存储和查询使用完全相同的 Embedding 模型

## 19. 多模态检索中图片描述不准确
**现象：** 生成的图片描述太泛泛，无法用于精确检索
**原因：** 图片描述 prompt 不够详细
**方案：** 使用结构化的图片描述模板（包含：主体、颜色、文字、布局、含义）

## 20. RAG 系统响应不稳定（有时好有时差）
**现象：** 同一个问题的回答质量时好时坏
**原因：** 文档检索结果波动，或 LLM 温度设置过高
**方案：** 设置固定随机种子，降低 temperature，缓存常见问题的检索结果
