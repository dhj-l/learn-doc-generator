第5章：工具（Tools）— 让代理拥有超能力

🎯 本章目标

🎯 本章学习目标
  • 理解工具的本质 —— 掌握 LangChain.js 中工具（Tool）的概念模型，明确工具是连接大语言模型与外部世界的桥梁，让模型从"只会说话"进化为"能做事"。
  • 熟练使用 tool() 函数 —— 能够使用 LangChain.js 提供的 tool() 工厂函数，从一个普通的 TypeScript 函数快速创建出模型可以调用的工具，并正确配置名称、描述与参数结构。
  • 掌握 Zod Schema 定义 —— 能够使用 Zod 库为工具的输入参数定义精确的类型约束与结构描述，确保模型传入的参数合法有效。
  • 灵活运用工具返回值 —— 了解工具可以返回字符串、对象、Command 以及 returnDirect 等不同类型的结果，并理解每种类型的适用场景。
  • 访问工具运行上下文 —— 学会通过 runtime 对象访问上下文配置、持久化存储、流式写入器等高级功能，为构建复杂工具奠定基础。
  • 构建多工具 Agent —— 能够将多个工具组合到一个 Agent 中，实现动态工具选择与权限过滤，最终搭建一个具备实际功能的多工具智能代理系统。

🔧 什么是工具

在现实世界中，一个人的能力不仅取决于他的知识储备，更取决于他能使用什么工具。一个医生如果只有医学知识但没有听诊器、手术刀和各种检测设备，那他的诊断和治疗能力将大打折扣。大语言模型（LLM）也是如此——虽然它拥有海量知识，但如果没有工具，它就无法与外部世界交互，无法获取实时数据，无法执行实际操作。

在 LangChain.js 中，工具（Tool）就是赋予 Agent 与外部世界交互能力的函数。工具就像给你的助手配备的各种专业设备——计算器、数据库终端、搜索引擎、文件系统、API 接口等等。Agent 通过工具来执行具体的任务：查询天气、搜索网络、读写文件、调用第三方 API、操作数据库……这些都是通过工具来实现的。

工具的核心组成

每一个工具都由以下三个核心部分组成：

  • 名称（name）：工具的唯一标识符，模型通过名称来识别和调用工具。按照惯例使用蛇形命名法（snake_case），例如 search_web、read_file、calculate_math。
  • 描述（description）：一段自然语言描述，告诉模型这个工具的用途和使用场景。模型会根据描述来决定何时调用哪个工具，因此描述的质量直接影响工具被正确调用的概率。
  • 输入模式（schema）：定义工具接受哪些输入参数、每个参数的类型和含义。LangChain.js 推荐使用 Zod 库来定义输入模式，这样可以在运行时自动进行参数验证。

为什么工具如此重要

没有工具的 LLM 就像一个只能在脑海中模拟的思考者。它可以推理、可以分析，但无法验证——它不知道今天北京的天气如何，不知道数据库里有什么数据，不知道最新的新闻是什么。工具的引入彻底改变了这一局面：

  • 实时性：通过搜索引擎或 API 调用获取最新数据，突破模型训练数据的时间限制。
  • 精确性：通过计算器和专用算法进行精确计算，避免模型"口算"时可能出现的错误。
  • 可操作性：通过数据库操作、文件读写、API 调用等工具，让模型不仅能"想"还能"做"。
  • 可靠性：通过 Zod Schema 的参数验证和错误处理机制，保证工具调用的可靠性和安全性。

⚡ tool() 函数详解

LangChain.js 提供了一个简洁而强大的工厂函数 tool()，用于将普通的 TypeScript/JavaScript 函数转换为模型可以调用的工具。这个函数是创建工具的主要方式，理解它的每一个参数都非常重要。

基本导入

TypeScript 导入 tool 函数 
// 从 @langchain/core/tools 导入 tool 工厂函数
import { tool } from "@langchain/core/tools";

// 通常还需要导入 Zod 来定义输入 schema
import { z } from "zod";

函数签名与参数说明

tool() 函数接受一个函数和一个配置对象作为参数，返回一个 Tool 实例。其核心参数如下：

TypeScript tool() 函数的完整参数 
const myTool = tool(
  // 第一个参数：工具实际执行的函数
  async (input, config) => {
    // input: 根据 schema 定义的类型化输入对象
    // config: 运行时配置，包含 callbacks 等信息
    return "工具返回的结果";
  },
  {
    // 第二个参数：工具的元数据配置对象

    // name: 工具名称，模型通过此名称调用工具
    // 惯例使用 snake_case 命名，简洁且具有描述性
    name: "my_custom_tool",

    // description: 工具功能描述，直接影响模型是否选择此工具
    // 描述应该清晰、完整，说明工具的用途、输入和输出
    description: "这是一个自定义工具，用于执行特定的任务。请在需要执行该任务时调用它。",

    // schema: 输入参数的 Zod Schema 定义
    // 用于验证和约束模型传入的参数
    schema: z.object({
      query: z.string().describe("搜索查询关键词"),
      limit: z.number().optional().describe("返回结果数量限制，默认为10"),
    }),
  }
);

参数详解

参数 类型 必填 说明
fn (input, config) => Promise<string> 是 工具执行的函数体。第一个参数是经过 schema 验证后的类型化输入对象，第二个参数是运行时配置。
name string 是 工具的唯一名称标识。建议使用 snake_case 命名，名称应简洁且能准确描述工具的功能。
description string 是 工具的自然语言描述。模型会根据此描述判断何时调用该工具，描述质量直接影响调用准确率。
schema ZodSchema 是 输入参数的 Zod Schema，用于类型推断和运行时验证。也可以是 JSON Schema 对象。

命名规范：蛇形命名法（snake_case）

在 LangChain.js 中，工具名称推荐使用蛇形命名法（snake_case），这是一种广泛采用的命名惯例。相比驼峰命名法（camelCase），蛇形命名法在工具场景中有几个优势：

  • 可读性更好：下划线分隔的名称在模型的 JSON 输出中更容易被正确解析，不会因为大小写问题引起混淆。
  • 兼容性更强：许多外部系统（如数据库、命令行工具）本身使用蛇形命名法，保持一致的命名风格可以降低认知负担。
  • 模型友好：大语言模型在训练数据中经常见到蛇形命名法，使用它可以提高模型正确生成工具调用名称的概率。
TypeScript 命名规范示例 
// ✅ 推荐：使用 snake_case
{ name: "search_web", ... }
{ name: "read_file", ... }
{ name: "query_database", ... }
{ name: "get_weather_forecast", ... }
{ name: "calculate_math_expression", ... }

// ❌ 不推荐：使用 camelCase 或其他命名方式
{ name: "searchWeb", ... }
{ name: "SearchWeb", ... }
{ name: "search-web", ... }  // 连字符在某些上下文中可能导致解析问题

第一个完整示例

下面是一个完整的工具创建示例，帮助你将所有参数串联起来理解：

TypeScript my-first-tool.ts 
import { tool } from "@langchain/core/tools";
import { z } from "zod";

// 创建一个简单的字符串反转工具
const reverseStringTool = tool(
  // 工具的执行函数
  async ({ text }) => {
    // 将输入字符串反转后返回
    const reversed = text.split("").reverse().join("");
    return `原始字符串: "${text}"\n反转结果: "${reversed}"`;
  },
  {
    // 使用 snake_case 命名
    name: "reverse_string",
    // 清晰的描述帮助模型理解何时使用此工具
    description: "将用户提供的字符串进行反转操作。当用户要求反转、倒序排列某个字符串时使用此工具。",
    // Zod schema 定义输入参数的结构和类型
    schema: z.object({
      text: z.string().describe("需要被反转的原始字符串"),
    }),
  }
);

// 工具创建完成后，可以绑定到模型上使用
import { ChatOpenAI } from "@langchain/openai";

const model = new ChatOpenAI({ modelName: "gpt-4o" });
const modelWithTools = model.bindTools([reverseStringTool]);

console.log("工具创建成功！");
console.log("工具名称:", reverseStringTool.name);
// 输出: 工具名称: reverse_string

📐 用 Zod 定义输入 Schema

在 LangChain.js 中，每个工具都需要定义一个输入模式（schema），用于描述工具接受哪些参数、每个参数的类型是什么、是否有默认值等信息。LangChain.js 推荐使用 Zod 库来定义这些 schema。

为什么选择 Zod

Zod 是一个 TypeScript 优先的 schema 声明和验证库。在 LangChain.js 生态中，选择 Zod 有以下几个重要原因：

  • 类型安全：Zod 可以从 schema 自动推断 TypeScript 类型，保证编译时类型检查和运行时验证的一致性。
  • 简洁的 API：Zod 提供了链式 API，代码可读性强，定义一个复杂的 schema 只需要几行代码。
  • 内置验证：Zod 在运行时自动进行数据验证，如果模型传入的参数不符合 schema 定义，会抛出明确的错误信息。
  • JSON Schema 转换：Zod schema 可以自动转换为 JSON Schema 格式，而 JSON Schema 是大语言模型理解工具参数的标准格式。
Bash 安装 Zod 
# 使用 npm 安装
npm install zod

# 或使用 yarn
yarn add zod

# 或使用 pnpm

常用 Zod 类型

Zod 提供了丰富的类型定义，以下是工具开发中最常用的类型：

TypeScript 常用 Zod 类型一览 
import { z } from "zod";

// 字符串类型
z.string()                     // 必填字符串
z.string().optional()          // 可选字符串，默认 undefined
z.string().default("默认值")    // 带默认值的字符串

// 数字类型
z.number()                     // 必填数字（整数或浮点数）
z.number().int()               // 必须是整数
z.number().min(0)              // 最小值为 0
z.number().max(100)            // 最大值为 100
z.number().positive()          // 必须为正数

// 布尔类型
z.boolean()                    // 必填布尔值
z.boolean().default(false)     // 默认为 false

// 枚举类型
z.enum(["small", "medium", "large"])  // 枚举值，只能是其中之一

// 数组类型
z.array(z.string())            // 字符串数组
z.array(z.number()).min(1)     // 至少包含一个数字的数组

// 联合类型
z.union([z.string(), z.number()])  // 可以是字符串或数字

使用 .describe() 为参数添加描述

.describe() 方法是 Zod schema 中最重要也是最容易被忽略的细节。它为每个参数添加自然语言描述，这些描述会被传递给语言模型，帮助模型理解每个参数的含义、格式要求和约束条件。描述的质量直接影响模型生成正确参数的能力。

TypeScript schema-with-describe.ts 
import { z } from "zod";

// ❌ 不好的做法：缺少 .describe() 描述
const badSchema = z.object({
  city: z.string(),
  unit: z.enum(["celsius", "fahrenheit"]),
  days: z.number(),
});
// 模型不知道 city 是什么格式（中文？英文？拼音？）
// 不知道 unit 是什么意思
// 不知道 days 是指未来几天

// ✅ 好的做法：每个字段都有清晰的 .describe()
const goodSchema = z.object({
  city: z
    .string()
    .describe("要查询天气的城市名称，使用中文，如 '北京'、'上海'、'广州'"),
  unit: z
    .enum(["celsius", "fahrenheit"])
    .optional()
    .default("celsius")
    .describe("温度单位，摄氏度(celsius)或华氏度(fahrenheit)，默认为摄氏度"),
  days: z
    .number()
    .int()
    .min(1)
    .max(7)
    .default(3)
    .describe("查询未来几天的天气预报，范围 1-7 天，默认为 3 天"),
});
// 模型现在能清楚地知道每个参数的含义和约束
⚠️ 重要提醒：永远不要省略 .describe() 方法。没有描述的参数会让模型"猜测"其含义，这会导致工具调用失败或传入错误的参数值。一个好的描述应该包含：参数的含义、可接受的值范围、格式示例以及默认行为。

完整的 Schema 定义示例

TypeScript complete-schema-example.ts 
import { z } from "zod";
import { tool } from "@langchain/core/tools";

// 一个复杂的数据库查询工具的 schema 定义
const queryDbSchema = z.object({
  table: z
    .string()
    .describe("要查询的数据表名称，例如 'users'、'orders'、'products'"),
  columns: z
    .array(z.string())
    .optional()
    .describe("要查询的列名数组，不传则查询所有列。示例: ['id', 'name', 'email']"),
  where: z
    .string()
    .optional()
    .describe("WHERE 条件表达式（SQL 语法），例如 \"age > 18 AND status = 'active'\""),
  orderBy: z
    .string()
    .optional()
    .describe("排序字段和方向，例如 'created_at DESC' 或 'name ASC'"),
  limit: z
    .number()
    .int()
    .min(1)
    .max(1000)
    .default(100)
    .describe("返回结果的最大行数，范围 1-1000，默认为 100"),
});

// TypeScript 会自动从 schema 推断出输入类型
// 类型为: { table: string; columns?: string[]; where?: string; orderBy?: string; limit: number }
type QueryDbInput = z.infer;

const queryDbTool = tool(
  async ({ table, columns, where, orderBy, limit }) => {
    // 在实际项目中，这里会连接数据库执行查询
    const cols = columns?.join(", ") || "*";
    const whereClause = where ? ` WHERE ${where}` : "";
    const orderClause = orderBy ? ` ORDER BY ${orderBy}` : "";
    const sql = `SELECT ${cols} FROM ${table}${whereClause}${orderClause} LIMIT ${limit}`;
    return `已执行查询: ${sql}\n结果: 模拟返回 ${limit} 条记录`;
  },
  {
    name: "query_database",
    description:
      "查询数据库中的数据。支持指定表名、列名、WHERE 条件、排序方式和结果数量限制。" +
      "适用于需要从数据库中检索信息的场景。返回查询结果的 JSON 格式数据。",
    schema: queryDbSchema,
  }
);

📦 工具返回值类型

工具执行完毕后，需要将结果返回给 Agent。LangChain.js 支持多种返回值类型，每种类型适用于不同的场景。理解这些类型的区别，可以帮助你构建更加灵活和强大的工具。

类型一：字符串返回（String）

最常见、最简单的返回类型。工具直接返回一个字符串，这个字符串会被封装为 ToolMessage 传递给模型。适用于大多数简单的工具场景。

TypeScript 字符串返回示例 
import { tool } from "@langchain/core/tools";
import { z } from "zod";

// 最简单的返回类型：返回字符串
const greetTool = tool(
  async ({ name }) => {
    // 直接返回字符串，会自动包装为 ToolMessage
    return `你好，${name}！欢迎使用 LangChain.js 教程系统。今天的学习进展非常顺利！`;
  },
  {
    name: "greet_user",
    description: "向指定用户发送问候语",
    schema: z.object({
      name: z.string().describe("用户的姓名"),
    }),
  }
);

类型二：对象返回（Object）

工具也可以返回一个对象。LangChain.js 会自动将对象序列化为 JSON 字符串。适用于返回结构化数据的场景。

TypeScript 对象返回示例 
// 返回对象类型 —— 会被自动序列化为 JSON 字符串
const getUserInfoTool = tool(
  async ({ userId }) => {
    // 模拟从数据库获取用户信息
    const user = {
      id: userId,
      name: "张三",
      email: "zhangsan@example.com",
      role: "开发者",
      lastLogin: "2026-06-08T10:30:00Z",
    };
    // 返回对象，LangChain.js 会自动将其 JSON.stringify
    return user;
  },
  {
    name: "get_user_info",
    description: "根据用户 ID 获取用户的详细信息",
    schema: z.object({
      userId: z.string().describe("用户的唯一标识符"),
    }),
  }
);

类型三：Command 返回

Command 是一种高级返回类型，允许工具对 Agent 的执行流程进行精确控制。通过 Command，工具可以更新 Agent 的状态、添加消息、甚至直接结束执行。这在需要工具对对话流程有更强控制力时非常有用。

TypeScript Command 返回示例 
import { tool } from "@langchain/core/tools";
import { Command } from "@langchain/langgraph";
import { z } from "zod";

// 返回 Command 类型，可以精确控制 Agent 的后续行为
const routeToHumanTool = tool(
  async ({ reason, priority }) => {
    // 返回一个 Command，更新 Agent 的状态并结束执行
    return new Command({
      update: {
        // 更新 Agent 状态中的值
        escalated: true,
        escalationReason: reason,
        priority: priority,
      },
      // 指定下一步操作（跳转到人工处理节点）
      goto: "human_review",
    });
  },
  {
    name: "escalate_to_human",

通常情况下，工具的返回值会经过模型的再次处理（总结、润色等）后才呈现给用户。但有时候你希望工具的结果直接返回给用户，跳过模型的二次处理。这时可以使用 returnDirect 选项。

📁 实战：文件读取工具

文件读取是 Agent 最常用的操作之一。下面的示例创建了一个安全的文件读取工具，它会读取指定路径的文件内容，并返回给模型进行分析。注意我们在工具中加入了路径安全检查，防止 Agent 访问敏感文件。

TypeScript file-read-tool.ts 
import { tool } from "@langchain/core/tools";
import { z } from "zod";
import * as fs from "fs/promises";
import * as path from "path";

// 定义允许访问的根目录，防止路径穿越攻击
const ALLOWED_ROOT = process.cwd();

const readFileTool = tool(
  async ({ filePath, encoding }) => {
    try {
      // 安全检查：解析并验证文件路径
      const resolvedPath = path.resolve(ALLOWED_ROOT, filePath);

      // 防止路径穿越——确保解析后的路径在允许的根目录下
      if (!resolvedPath.startsWith(ALLOWED_ROOT)) {
        return `安全错误：不允许访问根目录之外的文件。请求的路径 "${filePath}" 超出了允许的范围。`;
      }

      // 检查文件是否存在
      try {
        await fs.access(resolvedPath);
      } catch {
        return `文件未找到："${filePath}" 不存在或没有读取权限。`;
      }

      // 获取文件信息
      const stats = await fs.stat(resolvedPath);
      if (stats.isDirectory()) {
        // 如果是目录，列出目录内容
        const entries = await fs.readdir(resolvedPath);
        return `📁 "${filePath}" 是一个目录，包含以下文件和子目录：\n${entries.map((e) => `  - ${e}`).join("\n")}`;
      }

      // 限制读取文件大小，防止内存溢出（最大 1MB）
      const MAX_SIZE = 1024 * 1024;
      if (stats.size > MAX_SIZE) {
        return `文件过大："${filePath}" 的大小为 ${(stats.size / 1024).toFixed(1)}KB，超过了 1MB 的读取限制。请指定文件的具体行范围。`;
      }

      // 读取文件内容
      const content = await fs.readFile(resolvedPath, {
        encoding: encoding as BufferEncoding,
      });

      return [
        `📄 文件: ${filePath}`,
        `📏 大小: ${stats.size} 字节`,
        `📝 编码: ${encoding}`,
        `--- 文件内容开始 ---`,
        content,
        `--- 文件内容结束 ---`,
      ].join("\n");
    } catch (error) {
      return `读取文件时发生错误: ${error instanceof Error ? error.message : String(error)}`;
    }
  },
  {
    name: "read_file",
    description:
      "读取项目目录中的文件内容。支持文本文件（如 .ts、.json、.md、.txt 等）。" +
      "如果指定的路径是目录，则列出该目录下的文件和子目录。路径相对于项目根目录。",
    schema: z.object({
      filePath: z
        .string()
        .describe("文件路径（相对于项目根目录），例如 'src/index.ts' 或 'README.md'"),
      encoding: z
        .enum(["utf-8", "ascii", "base64"])
        .optional()
        .default("utf-8")
        .describe("文件编码格式，默认为 utf-8"),
    }),
  }
);

export { readFileTool };
⚠️ 安全警告：在生产环境中使用文件操作工具时，务必进行严格的路径验证和权限控制。永远不要让 Agent 可以任意读取服务器上的文件。上面的示例通过 ALLOWED_ROOT 限制了可访问的目录范围，这是最基本的安全措施。

🌐 实战：API 调用工具

在实际应用中，Agent 经常需要调用外部 REST API 来获取数据或执行操作。下面的示例创建了一个通用的 API 调用工具，可以向指定的 API 端点发送请求并返回结果。

TypeScript api-call-tool.ts 
import { tool } from "@langchain/core/tools";
import { z } from "zod";

// 定义 API 调用工具
const callApiTool = tool(
  async ({ url, method, headers, body, timeout }) => {
    try {
      // 构建请求选项
      const options: RequestInit = {
        method,
        headers: {
          "Content-Type": "application/json",
          ...Object.fromEntries(
            // 将头部字符串数组解析为键值对
            (headers || []).map((h) => {
              const [key, ...rest] = h.split(":");
              return [key.trim(), rest.join(":").trim()];
            })
          ),
        },
        // 设置请求超时时间
        signal: AbortSignal.timeout(timeout * 1000),
      };

      // 如果有请求体且不是 GET 请求，添加 body
      if (body && method !== "GET") {
        options.body = JSON.stringify(body);
      }

      console.log(`🌐 正在调用 API: ${method} ${url}`);

      // 发送请求
      const response = await fetch(url, options);
      const statusCode = response.status;

      // 尝试解析响应为 JSON
      let responseData;
      const contentType = response.headers.get("content-type");
      if (contentType?.includes("application/json")) {
        responseData = await response.json();
      } else {
        responseData = await response.text();
      }

      // 返回结构化的结果
      return JSON.stringify(
        {
          success: response.ok,
          statusCode,
          headers: Object.fromEntries(response.headers.entries()),
          data: responseData,
        },
        null,
        2
      );
    } catch (error) {
      // 处理各种错误情况
      if (error instanceof DOMException && error.name === "TimeoutError") {
        return `错误：API 调用超时（超过 ${timeout} 秒）。请检查 URL 是否正确或稍后重试。`;
      }
      return `错误：API 调用失败 — ${error instanceof Error ? error.message : String(error)}`;
    }
  },
  {
    name: "call_api",
    description:
      "调用外部 REST API。支持 GET、POST、PUT、DELETE 等 HTTP 方法。" +
      "返回 API 的响应状态码、响应头和响应体。适用于需要获取外部服务数据或执行远程操作的场景。",
    schema: z.object({
      url: z
        .string()
        .url()
        .describe("完整的 API 端点 URL，例如 'https://api.example.com/data'"),
      method: z
        .enum(["GET", "POST", "PUT", "DELETE", "PATCH"])
        .default("GET")
        .describe("HTTP 请求方法，默认为 GET"),
      headers: z
        .array(z.string())
        .optional()
        .describe("额外的请求头数组，格式为 'Key: Value'，例如 ['Authorization: Bearer token123']"),
      body: z
        .record(z.unknown())
        .optional()
        .describe("请求体对象（JSON 格式），仅对 POST/PUT/PATCH 方法有效"),
      timeout: z
        .number()
        .int()
        .min(1)
        .max(60)
        .default(10)
        .describe("请求超时时间（秒），范围 1-60，默认 10 秒"),
    }),
  }
);

export { callApiTool };

🎛️ 动态工具选择

在真实应用中，不同的用户可能有不同的权限级别。一个管理员可以调用所有工具，而普通用户可能只能调用部分工具。LangChain.js 允许你在运行时根据用户角色或权限动态过滤可用工具列表。

TypeScript dynamic-tool-selection.ts 
import { tool, type Tool } from "@langchain/core/tools";
import { z } from "zod";

// 定义工具的权限级别
type PermissionLevel = "public" | "user" | "admin";

// 自定义工具元数据扩展
interface ToolMetadata {
  tool: Tool;
  requiredPermission: PermissionLevel;
  category: string;
}

// 权限等级的数值映射，用于比较
const PERMISSION_RANK: Record<PermissionLevel, number> = {
  public: 0,
  user: 1,
  admin: 2,
};

// 创建一系列带权限标记的工具
const searchPublicDocsTool = tool(
  async ({ query }) => `搜索公开文档: "${query}" — 找到 5 条相关结果。`,
  {
    name: "search_public_docs",
    description: "搜索公开的文档和知识库",
    schema: z.object({
      query: z.string().describe("搜索关键词"),
    }),
  }
);

const manageUsersTool = tool(
  async ({ action, userId }) =>
    `管理操作执行: ${action} 用户 ${userId} — 操作成功。`,
  {
    name: "manage_users",
    description: "管理员功能：管理用户账户（启用、禁用、删除等）",
    schema: z.object({
      action: z.enum(["enable", "disable", "delete"]).describe("要执行的操作"),
      userId: z.string().describe("目标用户 ID"),
    }),
  }
);

const viewAnalyticsTool = tool(
  async ({ dateRange }) =>
    `分析数据 (${dateRange})：日活用户 1,234，新增用户 56，会话时长平均 8.5 分钟。`,
  {
    name: "view_analytics",
    description: "查看系统分析数据和统计报告",
    schema: z.object({
      dateRange: z.string().describe("时间范围，例如 'last7days'、'thisMonth'"),
    }),
  }
);

// 工具注册表：将工具与权限关联
const toolRegistry: ToolMetadata[] = [
  {
    tool: searchPublicDocsTool,
    requiredPermission: "public",
    category: "文档",
  },
  {
    tool: viewAnalyticsTool,
    requiredPermission: "user",
    category: "分析",
  },
  {
    tool: manageUsersTool,
    requiredPermission: "admin",
    category: "管理",
  },
];

// 根据用户权限级别过滤可用工具
function getToolsForUser(userPermission: PermissionLevel): Tool[] {
  const userRank = PERMISSION_RANK[userPermission];

  return toolRegistry
    .filter((entry) => PERMISSION_RANK[entry.requiredPermission] <= userRank)
    .map((entry) => entry.tool);
}

// 使用示例
const publicTools = getToolsForUser("public");
console.log(`公开用户可用工具: ${publicTools.map((t) => t.name).join(", ")}`);
// 输出: 公开用户可用工具: search_public_docs

const userTools = getToolsForUser("user");
console.log(`普通用户可用工具: ${userTools.map((t) => t.name).join(", ")}`);
// 输出: 普通用户可用工具: search_public_docs, view_analytics

const adminTools = getToolsForUser("admin");
console.log(`管理员可用工具: ${adminTools.map((t) => t.name).join(", ")}`);
// 输出: 管理员可用工具: search_public_docs, view_analytics, manage_users

通过这种模式，你可以将工具的定义和权限管理分离，让 Agent 的工具集真正实现动态化。在实际项目中，权限信息通常来自用户的 JWT Token 或 Session 数据，可以在创建 Agent 时根据当前用户的权限级别来组装对应的工具列表。

// ===== 工具1：天气查询 =====
const getWeatherTool = tool(
  async ({ city }) => {
    const weatherDb: Record<string, string> = {
      "北京": "晴天，28°C，湿度 45%，适合外出",
      "上海": "多云，26°C，湿度 72%，可能有阵雨",
      "广州": "雷阵雨，32°C，湿度 88%，建议带伞",
    };
    return weatherDb[city] || `暂时没有 ${city} 的天气数据。`;
  },
  {
    name: "get_weather",
    description: "查询指定城市的当前天气信息",
    schema: z.object({
      city: z.string().describe("城市名称，如 '北京'、'上海'"),
    }),
  }
);

// ===== 工具2：计算器 =====
const calculatorTool = tool(
  async ({ expression }) => {
    try {
      // 安全的数学表达式求值：只允许数字和基本运算符
      const sanitized = expression.replace(/[^0-9+\-*/().%\s]/g, "");
      if (sanitized !== expression) {
        return "错误：表达式包含不允许的字符。仅支持数字和运算符 (+, -, *, /, %, 括号)。";
      }
      // 使用 Function 构造器安全求值
      const result = new Function(`"use strict"; return (${sanitized})`)();
      return `${expression} = ${result}`;
    } catch {
      return `错误：无法计算表达式 "${expression}"，请检查语法是否正确。`;
    }
  },
  {
    name: "calculator",
    description: "计算数学表达式。支持加减乘除、括号和百分比运算。例如: '2 + 3 * 4' 或 '(100 - 20) * 0.8'",
    schema: z.object({
      expression: z.string().describe("要计算的数学表达式，如 '2 + 3 * 4'"),
    }),
  }
);

// ===== 工具3：读取项目文件 =====
const readFileTool = tool(
  async ({ filePath }) => {
    try {
      const content = await fs.readFile(filePath, "utf-8");
      // 限制返回长度，避免过长内容
      const truncated =
        content.length > 5000
          ? content.substring(0, 5000) + "\n\n... (文件内容过长，已截断)"
          : content;
      return `📄 ${filePath} 的内容:\n${truncated}`;
    } catch (error) {
      return `读取文件失败: ${error instanceof Error ? error.message : String(error)}`;
    }
  },
  {
    name: "read_file",
    description: "读取项目中的文件内容。适用于查看配置文件、源代码、文档等。",
    schema: z.object({
      filePath: z.string().describe("文件路径，如 'package.json' 或 'src/index.ts'"),
    }),
  }
);

// ===== 工具4：当前日期时间 =====
const getCurrentTimeTool = tool(
  async () => {
    const now = new Date();
    return [
      `当前日期和时间: ${now.toLocaleString("zh-CN", { timeZone: "Asia/Shanghai" })}`,
      `星期: ${["日", "一", "二", "三", "四", "五", "六"][now.getDay()]}`,
      `Unix 时间戳: ${Math.floor(now.getTime() / 1000)}`,
    ].join("\n");
  },
  {
    name: "get_current_time",
    description: "获取当前的日期、时间和星期信息。当用户询问今天是几号、现在几点等时间相关问题时使用。",
    schema: z.object({}),
  }
);

// ===== 创建多工具 Agent =====
async function main() {
  // 初始化语言模型
  const model = new ChatOpenAI({
    modelName: "gpt-4o",
    temperature: 0, // 使用确定性输出，提高工具调用的准确性
  });

  // 创建 Agent，注册所有工具
  const agent = createReactAgent({
    llm: model,
    tools: [getWeatherTool, calculatorTool, readFileTool, getCurrentTimeTool],
    // 可选：自定义系统提示
    prompt:
      "你是一个多才多艺的助手。你可以查询天气、进行数学计算、读取文件和获取当前时间。" +
      "请根据用户的问题，选择合适的工具来回答。回答时使用中文，保持友好和专业。",
  });

  // 测试1：查询天气
  console.log("===== 测试1: 天气查询 =====");
  const weatherResult = await agent.invoke({
    messages: [{ role: "user", content: "北京今天天气怎么样？" }],
  });
  console.log(
    "回答:",
    weatherResult.messages[weatherResult.messages.length - 1].content
  );

  // 测试2：数学计算
  console.log("\n===== 测试2: 数学计算 =====");
  const calcResult = await agent.invoke({
    messages: [{ role: "user", content: "帮我算一下 (123 + 456) * 789 等于多少" }],
  });
  console.log(
    "回答:",
    calcResult.messages[calcResult.messages.length - 1].content
  );

  // 测试3：多工具组合使用
  console.log("\n===== 测试3: 多工具组合 =====");
  const combinedResult = await agent.invoke({
    messages: [
      {
        role: "user",
        content: "现在几点了？另外帮我查一下上海的天气，再算一下 365 * 24 是多少。",
      },
    ],
  });
  console.log(
    "回答:",
    combinedResult.messages[combinedResult.messages.length - 1].content
  );
}

main().catch(console.error);
💡 关键要点
在这个综合示例中，Agent 拥有 4 个不同的工具。当你提出包含多个问题的请求时，模型会自动识别并调用所有需要的工具，然后将结果汇总成一个完整的回答。这就是多工具 Agent 的强大之处——它能根据用户的意图自主选择和组合工具。

🐛 常见错误

错误 1：Schema 类型与函数参数不匹配

症状：模型调用工具时传入的参数类型错误，例如期望字符串却收到数字，或者缺少必填参数。
原因：Zod schema 中的类型定义与函数体中实际使用的类型不一致。schema 是模型生成参数的"参考手册"，如果定义不准确，模型就无法生成正确类型的参数。
解决方案：确保 schema 中每个字段的类型、约束和描述与函数体的期望完全一致。始终使用 .describe() 明确说明参数的格式和取值范围。
TypeScript 错误示例 vs 正确示例 
// ❌ 错误：schema 定义 limit 为 string，但函数体期望 number
const badTool = tool(
  async ({ limit }) => {
    // 这里期望 limit 是数字，但 schema 定义为 string
    return Array(limit).fill("item"); // 运行时错误！
  },
  {
    name: "bad_example",
    description: "示例",
    schema: z.object({
      limit: z.string().describe("数量限制"),  // 应该是 z.number()
    }),
  }
);

// ✅ 正确：schema 类型与函数参数一致
const goodTool = tool(
  async ({ limit }) => {
    // limit 现在是 number 类型，与 schema 一致
    return Array(limit).fill("item");
  },
  {
    name: "good_example",
    description: "示例",
    schema: z.object({
      limit: z.number().int().min(1).max(100).describe("返回结果的数量限制，范围 1-100"),
    }),
  }
);

错误 2：工具名称冲突

症状：注册多个工具时出现错误，或 Agent 调用了错误的工具。模型的行为变得不可预测。
原因：多个工具使用了相同的 name 值。在 LangChain.js 中，工具名称是工具的唯一标识符，名称冲突会导致后注册的工具覆盖先注册的工具。
解决方案：确保每个工具都有唯一的名称。建议采用 动作_对象 的命名模式（如 search_users、delete_files），既能保证唯一性又便于理解。
TypeScript 避免名称冲突 
// ❌ 名称冲突！两个工具都叫 "search"
const searchUsersTool = tool(fn1, { name: "search", ... });
const searchDocsTool = tool(fn2, { name: "search", ... }); // 覆盖了第一个！

// ✅ 使用具体且唯一的名称
const searchUsersTool = tool(fn1, { name: "search_users", ... });
const searchDocsTool = tool(fn2, { name: "search_documents", ... });

错误 3：工具描述不够清晰

症状：Agent 在应该使用某个工具时没有使用，或者在不应该使用时错误地使用了该工具。模型的工具选择行为不符合预期。
原因：工具的 description 不够清晰或缺少关键信息。模型完全依赖描述来决定何时调用哪个工具，模糊的描述会导致模型做出错误的判断。
解决方案：为每个工具编写详细、明确的描述，包含：工具的用途、适用场景、输入要求、输出格式和限制条件。可以包含使用示例来帮助模型理解。
TypeScript 改善工具描述 
// ❌ 描述太模糊
{
  name: "search",
  description: "搜索",
  // 模型不知道搜索什么、搜索范围、返回格式
}

// ✅ 描述详细明确
{
  name: "search_documents",
  description:
    "在公司内部知识库中搜索文档。根据关键词进行全文检索，" +
    "返回最相关的文档标题、摘要和链接。" +
    "适用于：查找公司政策、技术文档、项目资料等内部信息。" +
    "不适用于：搜索互联网信息或查询实时数据。",
}

错误 4：异步操作未使用 await

症状：工具返回了 [object Promise] 字符串而不是实际结果，或者工具执行"成功"但返回值是未定义的。
原因：工具函数体中有异步操作（如 fetch 请求、文件读取等），但忘记在返回结果前使用 await 等待异步操作完成。
解决方案：确保工具函数声明为 async，并且所有异步操作都使用 await 关键字等待完成后再返回结果。
TypeScript 正确处理异步操作 
// ❌ 忘记 await，返回的是 Promise 而不是数据
const badApiTool = tool(
  async ({ url }) => {
    const response = fetch(url);          // 缺少 await！
    const data = response.json();         // 缺少 await！
    return data;                          // 返回 Promise 对象
  },
  { name: "bad_api", description: "...", schema: z.object({ url: z.string() }) }
);

// ✅ 正确使用 await
const goodApiTool = tool(
  async ({ url }) => {
    const response = await fetch(url);    // await 等待 fetch 完成
    const data = await response.json();   // await 等待 JSON 解析完成
    return JSON.stringify(data);          // 返回实际数据
  },
  { name: "good_api", description: "...", schema: z.object({ url: z.string() }) }
);

📝 本章小结

在本章中，我们深入学习了 LangChain.js 中工具（Tools）的核心概念和实践方法。以下是本章的关键知识点回顾：

  • 工具的本质：工具是连接大语言模型与外部世界的桥梁。它由名称、描述和输入模式三部分组成，让模型从"只会说话"进化为"能做事"。
  • tool() 函数：LangChain.js 提供的工厂函数，接受一个执行函数和一个配置对象，快速创建模型可调用的工具。工具名称推荐使用蛇形命名法（snake_case）。
  • Zod Schema：使用 Zod 定义工具的输入参数类型和约束。.describe() 方法至关重要，它为模型提供参数的自然语言描述，直接影响工具调用的准确性。
  • 四种返回值类型：字符串（最常见）、对象（自动 JSON 序列化）、Command（控制 Agent 流程）、returnDirect（直接输出给用户）。根据使用场景选择合适的返回类型。
  • 运行时上下文：通过 config 参数访问 configurable（只读配置）、store（持久化存储）、writer（流式输出）等运行时信息，支持构建复杂的高级工具。
  • 动态工具选择：可以根据用户权限、角色或其他运行时条件动态过滤可用工具列表，实现精细化的工具访问控制。
  • 多工具 Agent：将多个工具注册到 Agent 中，模型会根据用户请求自主选择和组合工具，实现复杂任务的自动化处理。
