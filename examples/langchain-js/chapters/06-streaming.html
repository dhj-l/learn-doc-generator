流式传输 Streaming — 实时响应

让用户体验从"等待焦虑"变为"实时互动"，掌握 LangChain.js 中最强大的交互增强技术

进阶 ⏱️ 预计学习时间：约 1 小时

🤔 为什么需要流式传输

想象你在餐厅点餐：如果厨师做完所有菜才一起端上来，你可能要饿等半小时。流式传输就像厨师做好一盘端一盘，你立刻就能开始享用。在 AI 应用的世界里，这个比喻再贴切不过了。

大语言模型（LLM）生成回复时，需要逐个 Token 进行预测和输出。一个完整的回复可能需要 3 到 15 秒才能生成完毕。如果没有流式传输，用户必须盯着空白屏幕等待全部内容生成完毕后才能看到任何东西——这种体验就是我们常说的"等待焦虑"。

传统模式 vs 流式模式

对比维度 传统阻塞模式 流式传输模式
首次响应时间 等待完整生成（3-15秒） 首个 Token 即刻显示（~200ms）
用户感知延迟 高，容易产生焦虑 低，交互感觉流畅自然
过程透明度 黑盒，用户不知道发生了什么 可实时展示 Agent 推理和工具调用过程
中断能力 无法中途取消 可随时中断生成过程
适用场景 后台任务、批量处理 聊天界面、实时助手、交互式应用
💡 核心理念
流式传输不仅仅是"快"——它让用户能够实时感知系统的运行状态。当你看到 Agent 正在"思考"、正在"调用工具"、正在"组织语言"，你就知道系统正在为你工作，而不是卡住了。这种透明度是构建可信 AI 应用的基石。

在 LangChain.js 中，流式传输是一个贯穿整个框架的核心特性。无论你使用的是简单的 ChatModel 对话，还是复杂的多步骤 Agent 工作流，都可以通过统一的 API 来获取实时数据流。LangChain v1.3 引入了全新的事件流式传输架构，让流式传输变得更加灵活和强大。

📡 事件流式传输（Event Streaming）

在 LangChain.js v1.3 中，推荐的流式传输方式是事件流式传输（Event Streaming）。这是一种基于 AsyncGenerator 的架构，能够以类型安全的方式实时获取 Agent 运行过程中的各种事件。

核心 API 概览

事件流式传输的核心方法是 agent.stream()，它返回一个异步生成器（AsyncGenerator），你可以用 for await...of 循环逐个处理事件。每个事件都有明确的类型信息，这就是所谓的类型投影（Typed Projection）。

TypeScript 
import { createAgent } from "langchain";

// 创建 Agent（与前面章节相同）
const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [weatherTool, calculatorTool],
});

// 使用 stream() 方法获取事件流
// 返回值是 AsyncGenerator，每个元素都是一个带类型的事件对象
const eventStream = agent.stream(
  { messages: [{ role: "user", content: "今天北京天气怎么样？" }] },
  { streamMode: "updates" }  // 指定流模式为 "updates"
);

// 用 for await...of 逐个处理事件
for await (const event of eventStream) {
  // 每个 event 都包含类型标记和对应的数据
  console.log("事件类型:", Object.keys(event));
  console.log("事件数据:", event);
}

类型投影机制

LangChain.js 的事件流式传输采用了类型投影设计。这意味着当你指定 streamMode 时，返回的事件类型会被精确映射到对应的 TypeScript 类型。例如，当你使用 "updates" 模式时，每个事件都对应 Agent 执行过程中的一个状态更新快照。使用 "messages" 模式时，事件则对应 LLM 输出的单个 Token。这种设计让你在编写代码时就能获得完善的类型检查和 IDE 自动补全支持，大幅提升了开发效率和代码安全性。

💡 与旧版 API 的区别
在 LangChain 早期版本中，流式传输主要通过 callbacks 回调实现（如 handleLLMNewToken）。v1.3 的事件流式传输是全新的推荐方式，它更直观、类型更安全、组合性更强。如果你在旧教程中看到基于回调的流式传输代码，建议迁移到新的事件流式传输 API。

🔧 Stream Modes 详解

LangChain.js 的流式传输支持多种模式（stream modes），每种模式专注于不同类型的数据流。你可以单独使用某一种模式，也可以同时组合多种模式来获取丰富的实时信息。下面我们逐一详细介绍。

5.1 updates — 状态更新流

当设置 streamMode: "updates" 时，Agent 每完成一个执行步骤后都会发射一个事件。这些事件构成了一条状态更新流，让你能够实时跟踪 Agent 的推理过程、工具调用情况和中间结果。

TypeScript 
// updates 模式：Agent 每步完成后发射事件
const stream = agent.stream(
  { messages: [{ role: "user", content: "计算 (15 + 27) * 3 的结果" }] },
  { streamMode: "updates" }
);

for await (const event of stream) {
  // event 的结构示例：
  // 当 LLM 生成回复时：{ agent: { messages: [...] } }
  // 当工具被调用时：  { tools: { messages: [...] } }
  // 当 Agent 结束时：  { agent: { messages: [...], returnValues: {...} } }

  if (event.agent) {
    console.log("[Agent] LLM 生成了新的消息");
    // 查看是否有工具调用
    const lastMsg = event.agent.messages[event.agent.messages.length - 1];
    if (lastMsg.tool_calls && lastMsg.tool_calls.length > 0) {
      console.log("[Agent] 决定调用工具:", lastMsg.tool_calls.map(tc => tc.name));
    }
  }

  if (event.tools) {
    console.log("[Tools] 工具执行完毕，返回结果");
    const toolMsg = event.tools.messages[event.tools.messages.length - 1];
    console.log("[Tools] 工具返回:", toolMsg.content);
  }
}

updates 模式非常适合用来在界面上显示 Agent 的工作进度。例如，你可以在用户界面中展示"正在思考..."、"正在查询天气工具..."、"正在整理回复..."等状态信息，让用户清楚地知道 Agent 正在做什么。

5.2 messages — Token 流

当设置 streamMode: "messages" 时，你会收到 LLM 实时输出的每一个 Token。这是实现打字机效果的标准方式——文字一个一个地出现在屏幕上，就像 Agent 正在实时打字一样。

TypeScript 
// messages 模式：逐 Token 流式输出
const stream = agent.stream(
  { messages: [{ role: "user", content: "用一段话介绍什么是量子计算" }] },
  { streamMode: "messages" }
);

let fullResponse = "";

for await (const [messageChunk, metadata] of stream) {
  // messages 模式的每个事件是一个元组 [messageChunk, metadata]
  // messageChunk: LLM 输出的 Token 片段
  // metadata: 包含消息来源等附加信息

  if (messageChunk.content) {
    const token = typeof messageChunk.content === "string"
      ? messageChunk.content
      : "";

    // 实时输出每个 Token（模拟终端打字效果）
    process.stdout.write(token);
    fullResponse += token;
  }
}

console.log("\n\n完整回复:", fullResponse);
📌 关于元组结构
messages 模式返回的每个事件是一个二元组（tuple），第一个元素是消息片段（AIMessageChunk），第二个元素是元数据对象（包含 langgraph_node 等运行时信息）。这种设计让你不仅能看到 Token 内容，还能知道这个 Token 是由哪个节点产生的。

5.3 custom — 自定义数据流

streamMode: "custom" 允许你在工具函数或 Agent 节点中发射任意自定义数据。这对于展示工具执行的中间进度特别有用。例如，你的工具可能需要处理大量数据，你可以通过自定义流来报告处理进度百分比。

TypeScript 
import { tool } from "langchain";
import { z } from "zod";

// 在工具中使用 config.writer 发射自定义数据
const dataAnalysisTool = tool(
  async (input, config) => {
    // 从 config 中获取 writer 函数
    const writer = config.writer;

    // 发射自定义进度信息
    writer?.({ step: "开始", progress: 0, message: "正在获取数据..." });

    // 模拟数据处理步骤
    const data = await fetchLargeDataset(input.query);

    writer?.({ step: "分析中", progress: 30, message: "数据获取完成，开始分析..." });

    const analysis = analyzeData(data);

    writer?.({ step: "分析中", progress: 70, message: "分析完成，正在生成报告..." });

    const report = generateReport(analysis);

    writer?.({ step: "完成", progress: 100, message: "报告生成完毕！" });

    return report;
  },
  {
    name: "data_analysis",
    description: "分析数据并生成报告",
    schema: z.object({
      query: z.string().describe("查询条件"),
    }),
  }
);

5.4 多模式组合

你可以将多种流模式组合在一起使用，同时获取不同类型的数据流。只需将 streamMode 设置为一个数组即可。这是构建丰富实时界面的推荐做法。

TypeScript 
// 同时使用 updates 和 messages 两种模式
const stream = agent.stream(
  { messages: [{ role: "user", content: "帮我分析一下特斯拉股票" }] },
  {
    streamMode: ["updates", "messages", "custom"],
    // 三种模式同时启用：
    // - "updates"  : 获取 Agent 步骤级状态更新
    // - "messages" : 获取逐 Token 输出
    // - "custom"   : 获取工具发出的自定义进度数据
  }
);

for await (const event of stream) {
  // 使用事件上的 streamMode 属性来区分事件类型
  switch (event.streamMode) {
    case "updates":
      console.log("[状态更新]", Object.keys(event.data));
      break;

    case "messages":
      const [chunk] = event.data;
      if (typeof chunk.content === "string") {
        process.stdout.write(chunk.content);
      }
      break;

    case "custom":
      console.log("[自定义]", event.data);
      break;
  }
}
Stream Mode 触发时机 事件内容 典型用途
"updates" Agent 每步完成后 包含 messages 的状态快照 展示 Agent 推理过程、工具调用进度
"messages" LLM 每输出一个 Token [AIMessageChunk, metadata] 元组 打字机效果、实时文本显示
"custom" 工具中调用 writer() 时 任意自定义数据对象 工具执行进度、中间计算结果

🛠️ 实战：流式输出 Agent 进度

让我们通过一个完整的实战案例来演示如何使用 updates 模式实时追踪 Agent 的工作流程。我们将构建一个可以查询天气和执行数学运算的 Agent，并在终端中实时展示它的推理和工具调用过程。

TypeScript 
import { createAgent } from "langchain";
import { tool } from "langchain";
import { z } from "zod";

// ===========================================
// 第一步：定义工具
// ===========================================

// 天气查询工具
const weatherTool = tool(
  async ({ city }) => {
    // 模拟天气 API 调用
    const weatherData: Record<string, { temp: number; desc: string }> = {
      "北京": { temp: 28, desc: "晴" },
      "上海": { temp: 32, desc: "多云" },
      "广州": { temp: 35, desc: "雷阵雨" },
    };

    const result = weatherData[city];
    if (result) {
      return `${city}当前天气：${result.desc}，气温 ${result.temp}°C`;
    }
    return `暂无 ${city} 的天气数据`;
  },
  {
    name: "get_weather",
    description: "查询指定城市的当前天气信息",
    schema: z.object({
      city: z.string().describe("城市名称，如 北京、上海"),
    }),
  }
);

// 数学计算工具
const calculatorTool = tool(
  async ({ expression }) => {
    // 安全的数学表达式计算（仅允许数字和基本运算符）
    const sanitized = expression.replace(/[^0-9+\-*/().]/g, "");
    try {
      const result = Function(`"use strict"; return (${sanitized})`)();
      return `计算结果：${expression} = ${result}`;
    } catch {
      return `计算错误：无法解析表达式 "${expression}"`;
    }
  },
  {
    name: "calculator",
    description: "执行数学计算，支持加减乘除和括号",
    schema: z.object({
      expression: z.string().describe("数学表达式，如 (15 + 27) * 3"),
    }),
  }
);

// ===========================================
// 第二步：创建 Agent 并启用流式传输
// ===========================================

const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [weatherTool, calculatorTool],
});

async function runWithProgress() {
  console.log("🤖 Agent 开始工作...\n");

  // 使用 updates 模式获取 Agent 的每步状态
  const stream = agent.stream(
    {
      messages: [
        { role: "user", content: "北京今天天气怎么样？如果气温乘以2是多少度？" },
      ],
    },
    { streamMode: "updates" }
  );

  let stepCount = 0;

  for await (const event of stream) {
    stepCount++;

    if (event.agent) {
      // ---- Agent 节点事件：LLM 进行推理 ----
      const messages = event.agent.messages;
      const lastMessage = messages[messages.length - 1];

      console.log(`📝 步骤 ${stepCount} [Agent - LLM 推理]`);

      // 检查 LLM 是否决定调用工具
      if (lastMessage.tool_calls && lastMessage.tool_calls.length > 0) {
        for (const tc of lastMessage.tool_calls) {
          console.log(`   🔧 决定调用工具: ${tc.name}`);
          console.log(`   📋 参数: ${JSON.stringify(tc.args)}`);
        }
      } else if (lastMessage.content) {
        // LLM 生成了最终文本回复
        console.log(`   💬 回复: ${lastMessage.content}`);
      }
    }

    if (event.tools) {
      // ---- Tools 节点事件：工具执行完毕 ----
      const messages = event.tools.messages;
      const toolResult = messages[messages.length - 1];

      console.log(`📝 步骤 ${stepCount} [工具执行完毕]`);
      console.log(`   ✅ 结果: ${toolResult.content}`);
    }

    console.log("   " + "─".repeat(50));
  }

  console.log(`\n✅ Agent 完成任务！共执行了 ${stepCount} 个步骤。`);
}

runWithProgress();
🤖 Agent 开始工作... 📝 步骤 1 [Agent - LLM 推理] 🔧 决定调用工具: get_weather 📋 参数: {"city":"北京"} ────────────────────────────────────────────────── 📝 步骤 2 [工具执行完毕] ✅ 结果: 北京当前天气：晴，气温 28°C ────────────────────────────────────────────────── 📝 步骤 3 [Agent - LLM 推理] 🔧 决定调用工具: calculator 📋 参数: {"expression":"28 * 2"} ────────────────────────────────────────────────── 📝 步骤 4 [工具执行完毕] ✅ 结果: 计算结果：28 * 2 = 56 ────────────────────────────────────────────────── 📝 步骤 5 [Agent - LLM 推理] 💬 回复: 北京今天天气晴朗，气温28°C。如果气温乘以2，就是56度。 ────────────────────────────────────────────────── ✅ Agent 完成任务！共执行了 5 个步骤。

从输出中可以清楚地看到 Agent 的完整推理链路：LLM 先决定查天气 → 天气工具返回结果 → LLM 再决定算乘法 → 计算器返回结果 → LLM 综合生成最终回复。整个过程就像看一位助手在你面前一步步操作，完全透明。

⚡ 实战：实时 Token 流式传输

接下来，我们使用 messages 模式来实现逐 Token 的流式输出。这种方式在 Web 应用中特别有价值——你可以将 Token 通过 WebSocket 或 Server-Sent Events 实时推送到前端，实现 ChatGPT 般的打字机效果。

TypeScript 
import { createAgent } from "langchain";

const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [],  // 纯聊天场景，不需要工具
});

async function streamTokens() {
  console.log("🤖 正在生成回复...\n");

  // 使用 messages 模式获取逐 Token 输出
  const stream = agent.stream(
    {
      messages: [
        {
          role: "user",
          content: "用三句话解释什么是机器学习，语言要通俗易懂",
        },
      ],
    },
    { streamMode: "messages" }
  );

  // 收集完整回复以统计信息
  let fullText = "";
  let tokenCount = 0;
  const startTime = Date.now();

  for await (const [messageChunk, metadata] of stream) {
    // messageChunk 是 AIMessageChunk 对象
    // metadata 包含 langgraph_node 等运行时信息

    // 只处理文本内容，过滤掉工具调用等非文本内容
    if (messageChunk.content && typeof messageChunk.content === "string") {
      const token = messageChunk.content;

      // 实时输出到终端（不换行，模拟打字效果）
      process.stdout.write(token);

      fullText += token;
      tokenCount++;
    }
  }

  const elapsed = Date.now() - startTime;
  const tokensPerSecond = ((tokenCount / elapsed) * 1000).toFixed(1);

  console.log("\n\n--- 生成统计 ---");
  console.log(`总 Token 数: ${tokenCount}`);
  console.log(`总耗时: ${elapsed}ms`);
  console.log(`生成速度: ${tokensPerSecond} tokens/s`);
}

streamTokens();
🤖 正在生成回复... 机器学习就像是教小孩认东西——你给电脑看成千上万张猫和狗的照片，它自己就能学会区分两者。它的核心是让机器从大量数据中找到规律，而不需要人类一步步编写具体的规则。当你在网上看到的推荐视频、手机上的人脸解锁，背后都是机器学习在发挥作用。 --- 生成统计 --- 总 Token 数: 87 总耗时: 2840ms 生成速度: 30.6 tokens/s

在 Web 应用中使用 Token 流

在实际的 Web 应用中，你通常会将 Token 流通过 HTTP 响应流发送给前端。以下是一个使用 Express 框架的示例：

TypeScript 
import express from "express";
import { createAgent } from "langchain";

const app = express();
const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [/* 你的工具列表 */],
});

// SSE（Server-Sent Events）流式端点
app.post("/api/chat", async (req, res) => {
  const { message } = req.body;

  // 设置 SSE 响应头
  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");
  res.setHeader("Connection", "keep-alive");

  const stream = agent.stream(
    { messages: [{ role: "user", content: message }] },
    { streamMode: "messages" }
  );

  try {
    for await (const [chunk, metadata] of stream) {
      if (chunk.content && typeof chunk.content === "string") {
        // 以 SSE 格式发送每个 Token
        res.write(`data: ${JSON.stringify({ token: chunk.content })}\n\n`);
      }
    }
    // 发送完成信号
    res.write(`data: ${JSON.stringify({ done: true })}\n\n`);
  } catch (error) {
    res.write(`data: ${JSON.stringify({ error: "生成失败" })}\n\n`);
  } finally {
    res.end();
  }
});

app.listen(3000, () => {
  console.log("流式 API 服务运行在 http://localhost:3000");
});
⚠️ 注意事项

在使用 SSE 或 WebSocket 发送 Token 流时，需要注意以下几点：

  • 确保在客户端断开连接时正确终止流，避免资源泄漏
  • 对 Token 内容进行 JSON 序列化以处理特殊字符（如引号、换行符）
  • 考虑添加超时机制，防止长时间无响应的请求占用服务器资源
  • 在生产环境中建议添加速率限制和并发控制

🎨 自定义流式更新

有时候你需要在工具执行过程中向用户报告进度信息。例如，一个搜索工具可能需要几秒钟来查询数据库，或者一个文件处理工具需要逐块读取和分析文件。通过 config.writer，你可以在工具函数内部发射任意自定义数据，前端可以接收这些数据并展示进度条、状态消息等。

TypeScript 
import { createAgent } from "langchain";
import { tool } from "langchain";
import { z } from "zod";

// ===========================================
// 带进度报告的文件分析工具
// ===========================================

const fileAnalysisTool = tool(
  async ({ filePath }, config) => {
    // 从 config 中获取 writer 函数
    // writer 会将数据发射到 "custom" 类型的流事件中
    const writer = config?.writer;

    // 第 1 步：读取文件
    writer?.({
      stage: "reading",
      progress: 10,
      message: `正在读取文件: ${filePath}`,
    });
    const content = await readFile(filePath);

    // 第 2 步：分析字数
    writer?.({
      stage: "analyzing",
      progress: 40,
      message: "正在统计字数和段落...",
    });
    const wordCount = content.split(/\s+/).length;
    const paragraphCount = content.split(/\n\n+/).length;

    // 第 3 步：提取关键词
    writer?.({
      stage: "extracting",
      progress: 70,
      message: "正在提取关键词...",
    });
    const keywords = extractKeywords(content);

    // 第 4 步：生成摘要
    writer?.({
      stage: "summarizing",
      progress: 90,
      message: "正在生成摘要...",
    });
    const summary = generateSummary(content);

    // 完成
    writer?.({
      stage: "complete",
      progress: 100,
      message: "分析完成！",
    });

    return JSON.stringify({
      wordCount,
      paragraphCount,
      keywords,
      summary,
    });
  },
  {
    name: "analyze_file",
    description: "分析文件内容，统计字数、提取关键词并生成摘要",
    schema: z.object({
      filePath: z.string().describe("要分析的文件路径"),
    }),
  }
);

// ===========================================
// 消费自定义流式数据
// ===========================================

const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [fileAnalysisTool],
});

async function analyzeWithProgress() {
  const stream = agent.stream(
    {
      messages: [
        { role: "user", content: "帮我分析 report.txt 文件的内容" },
      ],
    },
    { streamMode: ["updates", "custom"] }  // 同时获取更新和自定义数据
  );

  for await (const event of stream) {
    // 处理自定义进度数据
    if (event.streamMode === "custom") {
      const { stage, progress, message } = event.data;
      // 在终端中渲染进度条
      const bar = "█".repeat(Math.floor(progress / 5)) +
                  "░".repeat(20 - Math.floor(progress / 5));
      process.stdout.write(`\r[${bar}] ${progress}% ${message}`);
    }

    // 处理状态更新
    if (event.streamMode === "updates" && event.data.agent) {
      console.log(""); // 换行
      console.log("[Agent] 收到分析结果，正在组织回复...");
    }
  }

  console.log("\n✅ 分析完成！");
}

analyzeWithProgress();

通过 config.writer 发射的自定义数据会以 streamMode === "custom" 的形式出现在事件流中。你可以用它来报告任何你想要的中间状态——进度百分比、正在处理的文件名、数据库查询的中间结果等等。这为构建具有丰富反馈的应用打开了无限可能。

🧠 推理 Token 流式传输

部分模型（如 OpenAI o1、DeepSeek-R1 等）支持推理 Token（Reasoning Tokens）。这些 Token 是模型在生成最终回复之前进行"思考"的过程记录，通常以 contentBlocks 的形式出现在流中，类型标记为 "reasoning"。

在 messages 模式下，你可以通过过滤 contentBlocks 来单独提取推理过程，或者将其展示给用户以增强透明度。

TypeScript 
import { createAgent } from "langchain";

// 使用支持推理的模型
const agent = createAgent({
  model: "openai:o3-mini",  // 支持推理 Token 的模型
  tools: [],
});

async function streamWithReasoning() {
  const stream = agent.stream(
    {
      messages: [
        { role: "user", content: "一个水池有两个水管，A管每小时注入3升，B管每小时排出1升。水池容量是20升，从空池开始，多久能注满？" },
      ],
    },
    { streamMode: "messages" }
  );

  let isReasoning = false;
  let isFinalAnswer = false;
  let reasoningText = "";
  let answerText = "";

  for await (const [chunk, metadata] of stream) {
    if (typeof chunk.content === "string") {
      // 普通文本 Token（最终回复部分）
      if (!isFinalAnswer) {
        console.log("\n📝 模型回复：");
        isFinalAnswer = true;
      }
      process.stdout.write(chunk.content);
      answerText += chunk.content;
    }

    // 检查 contentBlocks 中是否有推理类型的内容
    // 某些模型的推理 Token 会以特殊块的形式出现
    if (Array.isArray(chunk.content)) {
      for (const block of chunk.content) {
        if (block.type === "reasoning") {
          if (!isReasoning) {
            console.log("🧠 模型推理过程：");
            isReasoning = true;
          }
          // 输出推理 Token
          if (block.text) {
            process.stdout.write(block.text);
            reasoningText += block.text;
          }
        }
      }
    }
  }

  console.log("\n\n--- 统计 ---");
  console.log(`推理部分长度: ${reasoningText.length} 字符`);
  console.log(`回复部分长度: ${answerText.length} 字符`);
}

streamWithReasoning();
📌 推理 Token 的显示策略

关于是否向用户展示推理 Token，取决于你的应用场景：

  • 教育/编程辅助场景：推荐展示推理过程，帮助用户理解模型的思考方式
  • 客服/对话场景：通常隐藏推理过程，只展示最终回复，保持界面简洁
  • 调试/开发场景：推荐展示，有助于发现和修复模型行为异常

无论哪种场景，你都可以通过过滤 contentBlocks 来灵活控制显示内容。

🚫 禁用流式传输

虽然流式传输通常是推荐的做法，但在某些场景下你可能需要禁用它。例如，在后台批处理任务中，你不需要实时输出；在单元测试中，你可能希望获得完整的确定性结果。LangChain.js 提供了简洁的方式来禁用流式传输。

TypeScript 
import { createAgent } from "langchain";

const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [/* 工具列表 */],
});

// 方式一：在调用时禁用流式传输
// 使用 invoke() 方法代替 stream()，直接获取完整结果
const result = await agent.invoke({
  messages: [{ role: "user", content: "你好" }],
});
// result 包含完整的最终状态，没有任何流式事件

// 方式二：通过配置参数禁用
const stream = agent.stream(
  { messages: [{ role: "user", content: "你好" }] },
  {
    streamMode: "updates",
    // 在特定模型节点上禁用流式传输
    // 这在某些不支持流式 API 的模型提供商中很有用
  }
);

// 方式三：在创建 Agent 时全局禁用
const batchAgent = createAgent({
  model: "openai:gpt-4o",
  tools: [],
  // 全局配置：禁用流式传输
  // 适用于纯后台批处理场景
});

// 使用 invoke() 获取完整结果（推荐的非流式调用方式）
const batchResult = await batchAgent.invoke({
  messages: [{ role: "user", content: "总结以下文本..." }],
});
console.log("完整结果:", batchResult);
💡 何时禁用流式传输
  • 后台批处理：批量处理大量请求时，不需要实时输出
  • 单元测试：测试时通常需要完整的确定性结果
  • 不支持流式的 API：某些第三方模型提供商可能不支持流式 API
  • 服务器资源有限：流式连接会占用更多并发资源，在资源受限时可考虑禁用
  • 简单的内部工具：不需要用户界面的内部自动化脚本

📝 自测题
问题 1：在 LangChain.js v1.3 中，推荐的流式传输 API 是什么？它与旧版基于 callbacks 的方式有什么区别？

推荐使用事件流式传输（Event Streaming）API，即 agent.stream() 方法配合 streamMode 参数。与旧版 callbacks 方式的主要区别：

  • 更直观：使用 for await...of 循环遍历事件，代码逻辑清晰
  • 类型安全：通过类型投影（Typed Projection）机制，根据 streamMode 返回精确的类型
  • 组合性强：可以同时组合多种 streamMode，如 ["updates", "messages"]
  • 无需手动注册回调：不需要在创建时传入复杂的 callbacks 对象
问题 2：streamMode: "updates" 和 streamMode: "messages" 的核心区别是什么？分别适用于什么场景？

"updates"：以 Agent 的执行步骤为粒度发射事件。Agent 每完成一个完整步骤（如 LLM 推理一步、执行一次工具调用）后，发射一个包含完整状态快照的事件。适用于展示 Agent 工作流程，如显示正在调用哪个工具、工具返回了什么结果等。

"messages"：以 Token 为粒度发射事件。LLM 每输出一个 Token，就立即发射一个事件，事件内容是 [AIMessageChunk, metadata] 元组。适用于实时文本输出（打字机效果），常见于聊天界面。

问题 3：如何在工具函数中发射自定义的流式进度数据？需要使用哪个 API？

在工具函数中，通过 config.writer 发射自定义数据。具体步骤：

  1. 工具函数的第二个参数 config 中包含 writer 函数
  2. 在工具执行过程中调用 writer(data) 发射任意数据
  3. 消费端在 streamMode 中包含 "custom"
  4. 自定义数据会以 event.streamMode === "custom" 的形式出现在事件流中
问题 4：如何同时获取 Agent 的状态更新（updates）和 Token 流（messages）？

将 streamMode 设置为包含多个模式的数组：

const stream = agent.stream(
  { messages: [...input] },
  { streamMode: ["updates", "messages", "custom"] }
);

然后在 for await 循环中通过 event.streamMode 属性来区分不同类型的事件，分别进行处理。这种方式可以让你同时展示 Agent 的工作进度和实时文本输出。

错误 1：混淆 invoke() 和 stream() 的返回值

TypeScript 
// ❌ 错误：把 stream() 的返回值当成普通对象使用
const result = agent.stream(
  { messages: [{ role: "user", content: "你好" }] },
  { streamMode: "updates" }
);
// result 是一个 AsyncGenerator，不是结果对象！
console.log(result.messages); // ❌ undefined — 没有 messages 属性

// ✅ 正确：使用 for await 循环遍历
const stream = agent.stream(
  { messages: [{ role: "user", content: "你好" }] },
  { streamMode: "updates" }
);

for await (const event of stream) {
  console.log(event); // ✅ 每个事件都包含完整的状态数据
}

// ✅ 或者使用 invoke() 获取完整结果（非流式）
const result = await agent.invoke({
  messages: [{ role: "user", content: "你好" }],
});
console.log(result.messages); // ✅ 直接访问完整的消息列表

错误 2：messages 模式下错误地访问事件内容

TypeScript

  console.log("来自节点:", metadata.langgraph_node);

TypeScript 
// ❌ 错误：没有错误处理，网络中断或 API 错误会直接崩溃
async function badStreamExample() {

  for await (const [chunk] of stream) {
    process.stdout.write(chunk.content); // 如果中途出错，进程直接崩溃

// ✅ 正确：添加 try-catch 和清理逻辑
async function goodStreamExample() {
  let stream;
  try {
    stream = agent.stream(input, { streamMode: "messages" });

    for await (const [chunk, metadata] of stream) {
      if (chunk.content && typeof chunk.content === "string") {
        process.stdout.write(chunk.content);
      }
    }
  } catch (error) {
    console.error("\n❌ 流式传输中断:", error.message);
    // 可以在这里添加重试逻辑或降级处理
  } finally {
    // 确保流被正确关闭，释放资源
    if (stream && typeof stream.return === "function") {
      await stream.return(undefined);
    }
    console.log("\n✅ 流已关闭");
  }
}
