🎯 综合实战：构建智能客服助手

将前九章所学融会贯通，从零搭建一个功能完整的 AI 智能客服系统

🎯 本章目标

恭喜你完成了前九章的学习！在本章中，我们将把 模型调用、消息处理、工具调用、流式传输、结构化输出、中间件、记忆系统 以及 Agent 编排 全部串联起来，构建一个真正具有实用价值的智能客服助手。这不是一个简单的 "Hello World" 示例——它涵盖了企业级客服系统所需的核心能力，包括产品查询、订单跟踪、退款处理、工单创建等完整业务流程。

通过本章的实战项目，你将深刻理解 LangChain.js 各组件之间如何协同工作，掌握从需求分析到代码落地的完整开发思路。这个项目可以直接作为你个人项目的基础框架，扩展后即可投入生产环境使用。

💡 学习建议

建议先完整阅读一遍本章内容，了解整体架构设计，然后再动手逐个步骤实现。遇到不理解的概念，可以回顾对应的前序章节。

📋 项目概述

我们要构建的是一个 智能客服助手（Smart Customer Service Assistant），它能够像一位训练有素的人工客服一样，与用户进行自然流畅的对话，并完成多种客服场景下的任务。下面是该助手需要具备的核心能力：

核心功能清单

  • 产品咨询 — 用户可以询问产品信息，助手会从模拟的产品数据库中检索相关数据并给出专业回答。例如："你们有哪些手机壳？"、"这款耳机支持蓝牙 5.3 吗？"
  • 订单查询 — 用户提供订单号后，助手可以查询订单状态、物流信息、预计送达时间等。例如："帮我查一下订单 ORD-001 的状态。"
  • 退款处理 — 用户发起退款请求时，助手会通过 人在环中（Human-in-the-Loop） 机制，在执行退款前请求人工审批，确保资金操作安全可控。
  • 记忆能力 — 助手能够记住当前会话的上下文信息（短期记忆），也能在多次会话间保持客户偏好等长期记忆数据。
  • 流式响应 — 对话内容以流式方式逐字推送给前端，提升用户交互体验，避免长时间等待。
  • 结构化工单 — 当需要记录客户问题时，助手可以通过结构化输出自动生成标准格式的工单数据，便于后端系统对接。
🔧 技术栈概览

本项目基于 LangChain.js v1.x 的 createAgent 架构，使用 TypeScript 编写，运行在 Node.js 环境中。所有数据均为内存模拟数据，无需外部数据库依赖，方便快速上手体验。

📝 需求分析

在动手写代码之前，我们先对项目进行系统化的需求分析，明确功能性需求和非功能性需求。

功能性需求

编号 需求描述 涉及组件 优先级
FR-01 用户能以自然语言与客服助手对话 模型 (Models)、消息 (Messages) P0
FR-02 助手能搜索并回答产品相关问题 工具 (Tools) — searchProducts P0
FR-03 助手能根据订单号查询订单状态 工具 (Tools) — checkOrderStatus P0
FR-04 助手能处理退款请求，但需要人工审批 工具 (Tools) — processRefund、人在环中 P0
FR-05 助手能自动创建标准格式的客服工单 结构化输出 (Structured Output) — createTicket P1
FR-06 助手在同一会话中能记住上下文 记忆系统 (Memory) — MemorySaver P0
FR-07 对话内容以流式方式返回 流式传输 (Streaming) P1
FR-08 记录每次工具调用的日志 中间件 (Middleware) — 日志中间件 P1
FR-09 对敏感操作进行安全校验 中间件 (Middleware) — 护栏中间件 P0
FR-10 助手能跨会话记住客户偏好 记忆系统 (Memory) — 自定义长期记忆 P2

非功能性需求

  • 可扩展性 — 代码结构清晰，方便后续新增工具和业务逻辑
  • 安全性 — 涉及资金操作时必须经过审批，防止误操作
  • 可维护性 — 关键逻辑有注释说明，易于团队协作和后期维护
  • 用户体验 — 流式响应提供即时反馈，避免用户感觉系统卡顿

架构的核心思想是 分层解耦：数据层负责模拟数据存储，工具层封装业务逻辑，记忆层管理上下文状态，中间件层提供横切关注点（日志、安全），Agent 层统一编排所有组件。每一层都可以独立替换和扩展，体现了良好的软件工程实践。

Step 2：定义系统提示词

系统提示词是智能客服助手的"灵魂"，它决定了助手的行为模式、语气风格和决策逻辑。一个好的系统提示词需要清晰地定义助手的角色、可用能力、行为规范和边界约束。

src/prompts/system-prompt.ts 

/**
 * 智能客服助手 — 系统提示词
 *
 * 定义了助手的角色定位、行为规范和能力边界。
 * 该提示词将作为 Agent 的 system message 传递给模型。
 */

export const SYSTEM_PROMPT = `你是一位专业、友善且高效的智能客服助手，服务于 "TechShop" 电子产品商城。

## 你的核心能力
1. **产品咨询**：你可以搜索产品数据库，回答用户关于产品规格、价格、库存等方面的问题。
2. **订单查询**：你可以根据订单号查询订单状态、物流信息和预计送达时间。
3. **退款处理**：你可以帮助用户发起退款请求，但退款操作必须经过人工审批后才能执行。
4. **工单创建**：当用户问题需要后续跟进时，你可以创建标准格式的客服工单。

## 行为规范
- 始终使用友好、专业的语气与用户交流。
- 回答要简洁明了，避免冗长的解释。
- 当用户表达不清楚时，主动追问以确认需求。
- 涉及退款等敏感操作时，务必向用户确认退款原因和订单信息。
- 如果用户的问题超出你的能力范围，坦诚告知并建议联系人工客服。
- 记住用户在对话中提到的偏好信息，提供个性化的服务体验。

## 重要约束
- 退款操作必须先获得用户确认，然后等待人工审批。
- 不要编造产品信息或订单信息，所有数据必须来自工具查询结果。
- 单笔退款金额不能超过 5000 元，超过需转接人工客服。
- 每个订单最多只能退款一次，请先检查是否已存在退款记录。

## 对话开场白
首次与用户对话时，使用以下开场白：
"您好！欢迎联系 TechShop 智能客服，我是您的专属助手。请问有什么可以帮到您的？"`;

/**
 * 工具选择引导提示 — 当用户意图不明确时，辅助模型做出正确的工具选择
 */
export const TOOL_GUIDANCE = `
## 工具使用指南
- 当用户提到"产品"、"价格"、"有没有"、"推荐"等关键词时 → 使用 searchProducts
- 当用户提到"订单"、"物流"、"发货"、"到哪了"等关键词时 → 使用 checkOrderStatus
- 当用户提到"退款"、"退货"、"不要了"等关键词时 → 使用 processRefund
- 当用户描述复杂问题需要记录跟进时 → 使用 createTicket
`;

系统提示词中包含了 角色定义、能力说明、行为规范 和 安全约束 四个关键部分。这种结构化的设计确保了助手在各种场景下都能做出合理、一致的响应。同时，工具选择引导提示可以帮助模型在意图不明确时更准确地匹配到正确的工具。

/**
 * 生成唯一 ID 的辅助函数
 */
export function generateId(prefix: string): string {
  const timestamp = Date.now().toString(36);
  const random = Math.random().toString(36).substring(2, 8);
  return `${prefix}-${timestamp}-${random}`.toUpperCase();
}

// ==================== 工具 3: 退款处理 ====================

export const processRefund = tool(
  async ({ orderId, reason, amount }) => {
    console.log(`  💰 [退款申请] 订单: ${orderId}, 原因: ${reason}, 金额: ¥${amount}`);

    // 查找订单
    const order = orders.find((o) => o.orderId === orderId);
    if (!order) {
      return `错误: 未找到订单 ${orderId}，无法处理退款。`;
    }

    // 检查是否已有退款记录
    const existingRefund = refundRecords.find((r) => r.orderId === orderId && r.status !== "rejected");
    if (existingRefund) {
      return `订单 ${orderId} 已存在退款申请（${existingRefund.refundId}），状态为 ${existingRefund.status}，不能重复申请。`;
    }

    // 检查退款金额是否超过限制
    if (amount > 5000) {
      return `退款金额 ¥${amount} 超过单笔上限 5000 元，请联系人工客服处理。`;
    }

    // 检查退款金额是否超过订单总额
    if (amount > order.totalAmount) {
      return `退款金额 ¥${amount} 超过订单总额 ¥${order.totalAmount.toFixed(2)}，请核实。`;
    }

    // 创建退款申请（状态为待审批）
    const refundId = generateId("REF");
    const refundRecord: RefundRecord = {
      refundId,
      orderId,
      amount,
      reason,
      status: "pending_approval",
      createdAt: new Date().toISOString(),
    };
    refundRecords.push(refundRecord);

    return [
      `✅ 退款申请已提交，等待人工审批。`,
      `退款单号: ${refundId}`,
      `订单号: ${orderId}`,
      `退款金额: ¥${amount.toFixed(2)}`,
      `退款原因: ${reason}`,
      `状态: 待审批（pending_approval）`,
      `注意: 退款将在人工审批通过后执行，预计 1-3 个工作日内到账。`,
    ].join("\n");
  },
  {
    name: "processRefund",
    description:
      "为用户提交退款申请。注意：退款不会立即执行，需要等待人工审批。请先确认用户确实需要退款，并获取退款原因。",
    schema: z.object({
      orderId: z.string().describe("要退款的订单号"),
      reason: z.string().describe("退款原因，例如：商品质量问题、不想要了、收到货不满意等"),
      amount: z.number().describe("退款金额（不能超过订单总额，且不能超过 5000 元）"),
    }),
  }
);

/**
 * 客服中间件
 *
 * 提供日志记录和安全护栏两个中间件函数。
 * 这些中间件在 Agent 的工具调用前后执行，实现非业务逻辑的横切关注点。

import type { AIMessage } from "@langchain/core/messages";

// ==================== 日志中间件 ====================

interface ToolCallLog {
  timestamp: string;
  toolName: string;
  args: Record<string, unknown>;
  result?: string;
  duration?: number;
}

const callLogs: ToolCallLog[] = [];

/**
 * 日志中间件 — 记录每次工具调用的详细信息
 *
 * 在工具调用前记录开始时间和参数，
 * 在工具调用后记录结果和耗时。
 */
export function loggingMiddleware(toolName: string, args: Record<string, unknown>): {
  onStart: () => ToolCallLog;
  onEnd: (result: string) => void;
} {
  return {
    onStart: () => {
      const log: ToolCallLog = {
        timestamp: new Date().toISOString(),
        toolName,
        args,
      };
      console.log(`\n📝 [日志] 工具调用开始:`);
      console.log(`   时间: ${log.timestamp}`);
      console.log(`   工具: ${toolName}`);
      console.log(`   参数: ${JSON.stringify(args, null, 2)}`);
      return log;
    },
    onEnd: (result: string) => {
      const lastLog = callLogs[callLogs.length - 1];
      if (lastLog) {
        lastLog.result = result.substring(0, 200) + (result.length > 200 ? "..." : "");
        lastLog.duration = Date.now() - new Date(lastLog.timestamp).getTime();
      }
      console.log(`   结果: ${result.substring(0, 100)}...`);
      console.log(`📝 [日志] 工具调用结束\n`);
    },
  };
}

/**
 * 获取所有调用日志（用于调试和审计）
 */
export function getCallLogs(): ToolCallLog[] {
  return [...callLogs];
}

// ==================== 护栏中间件 ====================

/**
 * 护栏中间件 — 对敏感操作进行安全校验
 *
 * 在执行退款等敏感操作前，检查参数的合法性和安全性。
 * 返回 { allowed, reason } 来决定是否允许执行。
 */
export function guardrailsMiddleware(
  toolName: string,
  args: Record<string, unknown>
): { allowed: boolean; reason?: string } {
  console.log(`🛡️ [护栏] 安全检查: ${toolName}`);

  // 退款操作的安全检查
  if (toolName === "processRefund") {
    const amount = args.amount as number;
    const orderId = args.orderId as string;
    const reason = args.reason as string;

    // 检查金额
    if (!amount || amount <= 0) {
      return { allowed: false, reason: "退款金额必须大于 0" };
    }

    // 检查金额上限
    if (amount > 5000) {
      return {
        allowed: false,
        reason: `退款金额 ¥${amount} 超过单笔上限 ¥5000，请联系人工客服`,
      };
    }

    // 检查退款原因
    if (!reason || reason.trim().length < 2) {
      return { allowed: false, reason: "退款原因不能为空或过短" };
    }

    // 检查订单号格式
    if (!orderId || !orderId.startsWith("ORD-")) {
      return { allowed: false, reason: "订单号格式不正确" };
    }

    console.log(`   ✅ 退款安全检查通过: 订单=${orderId}, 金额=¥${amount}`);
    return { allowed: true };
  }

  // 工单创建的安全检查
  if (toolName === "createTicket") {
    const subject = args.subject as string;

    if (!subject || subject.length < 2) {
      return { allowed: false, reason: "工单主题不能为空" };
    }

    if (subject.length > 200) {
      return { allowed: false, reason: "工单主题过长，不能超过 200 字符" };
    }

    console.log(`   ✅ 工单安全检查通过`);
    return { allowed: true };
  }

  // 其他操作默认放行
  console.log(`   ✅ 检查通过（非敏感操作）`);
  return { allowed: true };
}

import { HumanMessage } from "@langchain/core/messages";

// 会话配置（每个用户使用唯一的 thread_id）
const threadConfig = { configurable: { thread_id: "customer-session-001" } };

/**
 * 发送用户消息并以流式方式接收响应
 *
 * @param userMessage - 用户输入的消息
 */
async function chat(userMessage: string): Promise<void> {
  console.log(`\n${"=".repeat(60)}`);
  console.log(`👤 用户: ${userMessage}`);
  console.log(`${"=".repeat(60)}`);

  try {
    // 使用 stream 方法进行流式输出
    const stream = await agent.stream(
      { messages: [new HumanMessage(userMessage)] },
      {
        ...threadConfig,
        streamMode: "messages",  // 同时输出 token 级别的消息流
      }
    );

    // 收集最终的完整回复
    let fullResponse = "";

    for await (const chunk of stream) {
      // chunk 是一个数组，包含当前批次的事件
      if (Array.isArray(chunk)) {
        for (const event of chunk) {
          // 如果是 AI 消息的 token，实时打印
          if (event && typeof event === "object" && "content" in event) {
            const content = (event as any).content;
            if (typeof content === "string" && content) {
              process.stdout.write(content);
              fullResponse += content;
            }
          }
        }
      }
    }

    console.log("\n"); // 换行
    console.log(`📎 [调试] 完整回复长度: ${fullResponse.length} 字符`);
  } catch (error) {
    console.error("❌ 发生错误:", error);
  }
}

/**
 * 带状态更新的流式处理（跟踪工具调用）
 */
async function chatWithUpdates(userMessage: string): Promise<void> {
  console.log(`\n${"=".repeat(60)}`);
  console.log(`👤 用户: ${userMessage}`);
  console.log(`${"=".repeat(60)}`);

  const stream = await agent.stream(
    { messages: [new HumanMessage(userMessage)] },
    {
      ...threadConfig,
      streamMode: "updates",  // 输出 Agent 状态更新
    }
  );

  for await (const update of stream) {
    // 每次更新包含 Agent 的一个步骤
    for (const [nodeName, nodeOutput] of Object.entries(update)) {
      if (nodeName === "agent") {
        // Agent 节点的输出（模型推理结果）
        const messages = nodeOutput.messages;
        const lastMsg = messages[messages.length - 1];
        if (lastMsg?.content) {
          console.log(`🤖 [Agent] ${lastMsg.content}`);
        }
        if (lastMsg?.tool_calls?.length) {
          for (const tc of lastMsg.tool_calls) {
            console.log(`🔧 [工具调用] ${tc.name}(${JSON.stringify(tc.args)})`);
          }
        }
      } else if (nodeName === "tools") {
        // 工具节点的输出（工具执行结果）
        const messages = nodeOutput.messages;
        for (const msg of messages) {
          const preview = typeof msg.content === "string"
            ? msg.content.substring(0, 150)
            : JSON.stringify(msg.content).substring(0, 150);
          console.log(`  ⬅️ [工具结果] ${preview}...`);
        }
      }
    }
  }
}
💡 streamMode 详解

streamMode: "messages" 提供 token 级别的流式输出，适合前端逐字显示；streamMode: "updates" 提供 Agent 步骤级别的状态更新，适合监控和调试。你也可以组合使用：streamMode: ["messages", "updates"] 同时获取两种流。

import "dotenv/config";

// ==================== 5. 对话函数 ====================

/**
 * 发送消息并以流式方式接收 Agent 的回复
 *
 * @param userMessage 用户输入的消息文本
 * @param useUpdates 是否使用 updates 模式（默认使用 messages 模式）
 */
async function chat(userMessage: string, useUpdates = false): Promise<void> {
  const separator = "─".repeat(60);
  console.log(`\n${separator}`);
  console.log(`👤 用户: ${userMessage}`);
  console.log(separator);

  const streamMode = useUpdates ? "updates" : "messages";

  try {
    const stream = await agent.stream(
      { messages: [new HumanMessage(userMessage)] },
      { ...threadConfig, streamMode }
    );

    if (useUpdates) {
      // Updates 模式：显示 Agent 的推理和工具调用过程
      for await (const update of stream) {
        for (const [nodeName, nodeOutput] of Object.entries(update)) {
          if (nodeName === "agent") {
            const msgs = nodeOutput.messages;
            const lastMsg = msgs[msgs.length - 1];
            if (lastMsg?.tool_calls?.length) {
              for (const tc of lastMsg.tool_calls) {
                // 护栏检查
                const check = guardrailsMiddleware(tc.name, tc.args);
                if (!check.allowed) {
                  console.log(`🛡️ [护栏拦截] ${check.reason}`);
                  continue;
                }
                // 日志记录
                loggingMiddleware(tc.name, tc.args);
                console.log(`🔧 [调用工具] ${tc.name}`);
              }
            }
            if (lastMsg?.content) {
              console.log(`🤖 [Agent] ${lastMsg.content}`);
            }
          } else if (nodeName === "tools") {
            const msgs = nodeOutput.messages;
            for (const msg of msgs) {
              const preview = String(msg.content).substring(0, 200);
              console.log(`  ➡️ [工具结果] ${preview}...`);
            }
          }
        }
      }
    } else {
      // Messages 模式：逐字流式输出最终回复
      let fullResponse = "";
      for await (const chunk of stream) {
        if (Array.isArray(chunk)) {
          for (const event of chunk) {
            if (event?.content && typeof event.content === "string") {
              process.stdout.write(event.content);
              fullResponse += event.content;
            }
          }
        }
      }
      console.log();
    }
  } catch (error) {
    console.error("❌ 对话出错:", error);
  }
}

下一步学习路径

完成了本教程的基础学习后，你可以根据自己的兴趣和项目需求，继续深入探索以下方向：

1. Deep Agents — 更强大的智能体

@langchain/langgraph 提供了更高级的 Agent 架构，包括支持多 Agent 协作的 Supervisor 模式、可以规划并执行复杂任务的 Plan-and-Execute 模式，以及支持 Agent 之间相互委派任务的 Handoff 机制。这些高级模式适合构建更复杂的业务系统。

2. LangGraph — 复杂工作流编排

当业务逻辑不再是简单的"推理-行动"循环时，你需要 LangGraph 的图编排能力。它允许你定义有状态的工作流，支持条件分支、并行执行、循环控制等复杂模式。例如，一个完整的客服系统可能需要"分类-路由-处理-审核-回复"的多步工作流。

3. LangSmith — 生产环境监控

将应用部署到生产环境后，你需要 LangSmith 来监控 Agent 的运行状态。LangSmith 提供了请求追踪（Trace）、性能监控、质量评估、调试回放等功能，帮助你持续优化 Agent 的表现。设置非常简单，只需在环境变量中配置 LANGSMITH_API_KEY 即可自动上报。

4. RAG — 基于知识库的应用

检索增强生成（Retrieval-Augmented Generation） 是让 AI 助手拥有"专业知识"的关键技术。通过将企业文档、产品手册、FAQ 等知识库内容向量化存储，Agent 可以在回答用户问题前先检索相关资料，从而给出更准确、更有依据的回答。在客服场景中，RAG 可以显著提升回答的专业度和可靠性。
