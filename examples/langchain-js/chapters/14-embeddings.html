🧬 Embedding Models — 文本向量化与语义表示

将文字转化为数学向量，让计算机理解语义的"距离"

💡 核心概念

概念一：文本向量化 — 把文字变成数学

文本向量化（Embedding） 是将自然语言文本转化为一组浮点数向量的过程。 这组向量捕捉了文本的语义信息——意思相近的文本在向量空间中彼此靠近， 意思不同的文本则相距较远。

💡 直观理解

想象一下，所有中文词汇都被映射到一个高维空间中。"猫"和"狗"的向量距离很近（都是宠物）， 而"猫"和"操作系统"的向量距离则很远。Embedding 模型的任务就是学习这种语义映射关系。

余弦相似度（Cosine Similarity）

衡量两个向量相似度的最常用方法是余弦相似度，计算公式为：

数学公式 
cosine_similarity(A, B) = (A · B) / (||A|| × ||B||)

其中：
  A · B   = A 和 B 的点积（对应元素相乘再求和）
  ||A||   = A 的模（向量长度）
  ||B||   = B 的模

值域：[-1, 1]
  1   → 方向完全相同（语义高度相似）
  0   → 正交（语义无关）
  -1  → 方向相反（语义相反）

向量维度（Dimensions）

向量的维度决定了模型能捕捉的语义精细程度：

维度代表模型特点
256 ~ 384Ollama (nomic-embed-text), text-embedding-3-small轻量、速度快、成本低，适合简单分类
768 ~ 1024text-embedding-ada-002, BGE-large均衡性能，通用场景
1536 ~ 3072text-embedding-3-large高精度，适合深度语义理解

概念二：OpenAIEmbeddings — 核心 API

LangChain.js 通过 OpenAIEmbeddings 类封装了 OpenAI 的 Embedding API。 它提供了两个核心方法：

方法用途输入输出
embedQuery(text)对查询文本进行向量化单个字符串一维数组 number[]
embedDocuments(docs[])对文档集合进行批量向量化字符串数组二维数组 number[][]
⚠️ embedQuery vs embedDocuments 的区别

embedQuery 专为用户查询设计——它可能使用不同的模型配置（如更短的输入截断）， 而 embedDocuments 用于知识库文档的批量嵌入。 在语义搜索中，通常先用 embedDocuments 处理所有文档并存入向量库， 用户查询时再调用 embedQuery 将问题向量化进行检索。

TypeScript 
import { OpenAIEmbeddings } from "@langchain/openai";

// ==========================================
// 初始化 Embedding 模型
// ==========================================
const embeddings = new OpenAIEmbeddings({
  model: "text-embedding-3-small",   // 默认模型
  dimensions: 256,                    // 可降低维度以节省成本
  openAIApiKey: process.env.OPENAI_API_KEY,
});

// ==========================================
// 1. 单条查询向量化
// ==========================================
const queryVector = await embeddings.embedQuery(
  "什么是机器学习？"
);
console.log(`向量维度：${queryVector.length}`);   // 256
console.log(`前 5 个值：${queryVector.slice(0, 5)}`);

// ==========================================
// 2. 批量文档向量化
// ==========================================
const docs = [
  "机器学习是人工智能的一个分支",
  "深度学习使用神经网络进行学习",
  "自然语言处理让计算机理解语言",
];

const docVectors = await embeddings.embedDocuments(docs);
console.log(`文档数量：${docVectors.length}`);     // 3
console.log(`向量维度：${docVectors[0].length}`);  // 256

概念三：Embedding 模型选型

选择合适的 Embedding 模型需要在维度、成本、性能之间做权衡：

模型维度价格（每 1K tokens）最大输入适用场景
text-embedding-3-small512（可降维）$0.000028191 tokens通用场景，性价比最高
text-embedding-3-large3072（可降维）$0.000138191 tokens高精度搜索，语义理解
text-embedding-ada-0021536（固定）$0.000108191 tokens遗留系统兼容
Ollama (nomic-embed-text)768免费（本地）2048 tokens本地开发、离线场景
💡 降维技巧

OpenAI 的 text-embedding-3 系列支持降维——你可以在创建时指定 dimensions 参数 （如 256 或 512），模型会返回较短但依然保留大部分语义信息的向量。 降维后存储成本更低，检索速度更快，且精度损失通常很小。

概念四：批量处理策略

当需要向量化的文档数量较大时，合理的批量处理策略能显著提升效率：

  • 批大小控制：OpenAI 单次调用最大可以处理 2048 个文档，但推荐每批 100~500 个以平衡速度和稳定性
  • 速率限制：注意 OpenAI API 的 RPM（每分钟请求数）和 TPM（每分钟 Token 数）限制
  • 错误重试：实现指数退避重试机制，应对限流错误
  • 并发控制：使用 p-limit 或类似库控制并发请求数

🔨 实战演练

实战一：文本向量化与相似度计算

场景描述：你有一组关于 AI 主题的句子，需要计算它们之间的语义相似度， 以便将相似主题的句子自动归类。

你的任务：使用 OpenAIEmbeddings 将句子向量化，然后计算两两之间的余弦相似度。

🧑‍💻 先自己写，写完再展开看参考答案
TypeScript 
import { OpenAIEmbeddings } from "@langchain/openai";

// ==========================================
// 工具函数：计算余弦相似度
// ==========================================
function cosineSimilarity(a: number[], b: number[]): number {
  if (a.length !== b.length) {
    throw new Error("向量维度不匹配");
  }

  let dotProduct = 0;
  let normA = 0;
  let normB = 0;

  for (let i = 0; i < a.length; i++) {
    dotProduct += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }

  const magnitude = Math.sqrt(normA) * Math.sqrt(normB);
  return magnitude === 0 ? 0 : dotProduct / magnitude;
}

// ==========================================
// 主程序
// ==========================================
async function main() {
  const embeddings = new OpenAIEmbeddings({
    model: "text-embedding-3-small",
  });

  const sentences = [
    "机器学习是人工智能的重要分支",
    "深度学习利用神经网络进行训练",
    "自然语言处理让计算机理解人类语言",
    "今天天气很好适合出去散步",
    "监督学习需要标注数据进行训练",
  ];

  // 1. 批量向量化
  console.log("🔄 正在向量化句子...");
  const vectors = await embeddings.embedDocuments(sentences);

  // 2. 计算相似度矩阵
  console.log("\n📊 相似度矩阵：");
  for (let i = 0; i < sentences.length; i++) {
    for (let j = i + 1; j < sentences.length; j++) {
      const sim = cosineSimilarity(vectors[i], vectors[j]);
      console.log(`[${i}] vs [${j}] ${sim.toFixed(4)}  |  "${sentences[i].slice(0, 20)}..." ~ "${sentences[j].slice(0, 20)}..."`);
    }
  }

  // 3. 找最相似的句子对
  let maxSim = -1;
  let bestPair = [0, 0];
  for (let i = 0; i < vectors.length; i++) {
    for (let j = i + 1; j < vectors.length; j++) {
      const sim = cosineSimilarity(vectors[i], vectors[j]);
      if (sim > maxSim) {
        maxSim = sim;
        bestPair = [i, j];
      }
    }
  }

  console.log(`\n🏆 最相似的句子对：`);
  console.log(`  句子 A: "${sentences[bestPair[0]]}"`);
  console.log(`  句子 B: "${sentences[bestPair[1]]}"`);
  console.log(`  相似度: ${maxSim.toFixed(4)}`);
}

main();

实战二：简单语义搜索

场景描述：你有一份简单的文档知识库，用户输入一个查询， 系统需要从文档中找到语义上最匹配的结果返回。

你的任务：实现一个完整的"查询 → 向量化 → 相似度排序 → 返回结果"的语义搜索流程。

🧑‍💻 先自己写，写完再展开看参考答案
TypeScript 
import { OpenAIEmbeddings } from "@langchain/openai";

// ==========================================
// 余弦相似度工具函数
// ==========================================
function cosineSimilarity(a: number[], b: number[]): number {
  let dotProduct = 0, normA = 0, normB = 0;
  for (let i = 0; i < a.length; i++) {
    dotProduct += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }
  return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
}

// ==========================================
// 简单的语义搜索引擎
// ==========================================
class SimpleSemanticSearch {
  private embeddings: OpenAIEmbeddings;
  private documents: string[] = [];
  private vectors: number[][] = [];

  constructor() {
    this.embeddings = new OpenAIEmbeddings({
      model: "text-embedding-3-small",
    });
  }

  // 添加文档并建立索引
  async addDocuments(docs: string[]): Promise {
    this.documents = docs;
    console.log(`📄 正在索引 ${docs.length} 个文档...`);
    this.vectors = await this.embeddings.embedDocuments(docs);
    console.log("✅ 索引完成！");
  }

  // 搜索最相关的文档
  async search(query: string, topK: number = 3): Promise<{ text: string; score: number }[]> {
    console.log(`🔍 搜索: "${query}"`);

    // 1. 将查询向量化
    const queryVector = await this.embeddings.embedQuery(query);

    // 2. 计算与所有文档的相似度
    const scores = this.vectors.map((docVector, i) => ({
      text: this.documents[i],
      score: cosineSimilarity(queryVector, docVector),
    }));

    // 3. 按相似度降序排列
    scores.sort((a, b) => b.score - a.score);

    // 4. 返回 topK 结果
    return scores.slice(0, topK);
  }
}

// ==========================================
// 测试
// ==========================================
async function main() {
  const searchEngine = new SimpleSemanticSearch();

  // 添加知识库文档
  await searchEngine.addDocuments([
    "LangChain 是一个用于构建 LLM 应用的框架，支持 JavaScript 和 Python",
    "Embedding 模型将文本转化为向量，用于语义搜索和相似度计算",
    "向量数据库（如 Chroma、Pinecone）专门存储和检索高维向量",
    "RAG（检索增强生成）结合了搜索和生成，提升回答的准确性",
    "Prompt Templates 帮助结构化和复用提示词",
    "Agent 可以自主调用工具来完成复杂任务",
    "Streaming 允许 LLM 逐字输出结果，提升用户体验",
    "LCEL 是 LangChain 的声明式链式组合语法",
  ]);

  // 执行搜索
  const results = await searchEngine.search("什么是向量数据库？");

  console.log("\n📋 搜索结果：");
  results.forEach((r, i) => {
    console.log(`  ${i + 1}. [${(r.score * 100).toFixed(1)}%] ${r.text}`);
  });

  console.log("\n--- 第二次搜索 ---");
  const results2 = await searchEngine.search("如何构建 LLM 应用？");
  results2.forEach((r, i) => {
    console.log(`  ${i + 1}. [${(r.score * 100).toFixed(1)}%] ${r.text}`);
  });
}

main();

技巧一：Ollama 本地 Embedding

如果你不想依赖 OpenAI 的云端 API，可以使用 Ollama 在本地运行 Embedding 模型。 Ollama 提供了 nomic-embed-text、all-minilm 等轻量模型， 完全离线且免费，非常适合开发和测试场景。

技巧二：批量 Embedding 策略

当需要处理大量文档时，合理的批量策略可以显著提升效率和稳定性：

TypeScript 
import { OpenAIEmbeddings } from "@langchain/openai";

// ==========================================
// 批量嵌入带重试和并发控制
// ==========================================

const embeddings = new OpenAIEmbeddings({
  model: "text-embedding-3-small",
  maxRetries: 3,            // 失败重试次数
  maxConcurrency: 5,        // 最大并发请求数
});

/**
 * 将大型文档列表分批向量化
 */
async function batchEmbed(
  documents: string[],
  batchSize: number = 100
): Promise {
  const allVectors: number[][] = [];

  for (let i = 0; i < documents.length; i += batchSize) {
    const batch = documents.slice(i, i + batchSize);
    console.log(`⏳ 处理第 ${i + 1}-${Math.min(i + batchSize, documents.length)} / ${documents.length} 个文档...`);

    try {
      const vectors = await embeddings.embedDocuments(batch);
      allVectors.push(...vectors);
    } catch (error) {
      console.error(`❌ 批次 ${i / batchSize + 1} 处理失败:`, error);
      // 可以在此实现指数退避重试
      throw error;
    }
  }

  return allVectors;
}

// ==========================================
// 使用示例
// ==========================================
async function main() {
  // 模拟 500 个文档
  const allDocs = Array.from(
    { length: 500 },
    (_, i) => `这是第 ${i + 1} 篇关于机器学习的文档内容...`
  );

  console.log("🚀 开始批量向量化...");
  const startTime = Date.now();

  const vectors = await batchEmbed(allDocs, 100);

  const elapsed = ((Date.now() - startTime) / 1000).toFixed(1);
  console.log(`✅ 完成！${vectors.length} 个向量，耗时 ${elapsed} 秒`);
  console.log(`向量维度: ${vectors[0]?.length ?? "N/A"}`);
}

main();

🧠 知识检查点

📝 自测题：检查你对本章内容的理解
1. 什么是文本向量化？余弦相似度的值域和含义是什么？

文本向量化是将自然语言转换为浮点数向量的过程。余弦相似度衡量两个向量的方向一致性，值域为 [-1, 1]：1 表示方向完全相同（语义高度相似），0 表示正交（语义无关），-1 表示方向相反（语义相反）。公式为：cos(A, B) = (A·B) / (||A|| × ||B||)。

2. embedQuery() 和 embedDocuments() 有什么区别？分别在什么时候使用？

embedQuery() 接收单个字符串，用于向量化用户查询（搜索时的问题），可能使用不同的截断策略。embedDocuments() 接收字符串数组，用于批量处理知识库文档。在语义搜索流程中，先用 embedDocuments 预处理全部文档并存入向量库，用户查询时调用 embedQuery 将问题向量化后进行检索。

3. 为什么说向量维度越高不一定越好？如何选择合适的维度？

更高维度虽然能捕捉更精细的语义，但会带来：① 更大的存储开销；② 更慢的检索速度（维度灾难）；③ 更高的 API 调用成本。选择合适的维度需要考虑：① 任务精度要求（简单分类 256 维足够，深度语义搜索 1024+）；② 向量数据库的限制和计费方式；③ OpenAI 的 text-embedding-3 系列支持降维，可在精度和效率间灵活权衡。

4. 批量处理大量文档时应该注意哪些问题？

① 批大小控制：推荐每批 100~500 个文档，避免单次超限；② 速率限制：注意 OpenAI 的 RPM/TPM 限制，使用 maxConcurrency 控制并发；③ 错误重试：实现指数退避重试机制处理 429 限流错误；④ 进度监控：对大量文档分批处理并记录进度；⑤ 断点续传：在工程化场景中保存已处理结果，避免重复调用。

5. Ollama 本地 Embedding 的优势和局限是什么？

优势：免费、离线运行、数据不出本机、适合开发和测试、无速率限制。局限：模型能力弱于 OpenAI（语义理解精度较低）、支持的最大输入令牌数较少（如 nomic-embed-text 仅 2048 tokens）、对硬件有要求（需要足够的 RAM/VRAM）。建议在开发阶段和敏感数据场景使用 Ollama，在生产级高精度场景使用 OpenAI。

🐛 常见错误

❌ 错误 1：API Key 未配置

症状：AuthenticationError: Incorrect API key provided

原因：没有设置 OPENAI_API_KEY 环境变量，或提供的 Key 无效。

解决：在 .env 文件中配置 OPENAI_API_KEY=sk-xxxx，或通过构造函数传入 openAIApiKey 参数。

❌ 错误 2：向量维度不匹配

症状：计算相似度时抛出 Error: 向量维度不匹配

原因：使用不同 Embedding 模型产生的向量维度不同，不能直接比较。

解决：确保查询和文档使用同一个 Embedding 模型，且 dimensions 参数一致。文档预处理时使用的模型必须与查询时使用的模型相同。

❌ 错误 3：输入文本超过最大 Token 限制

症状：InvalidRequestError: total tokens exceeds maximum

原因：输入文本过长，超过了 Embedding 模型的最大输入长度（如 8191 tokens）。

解决：在向量化之前对长文档进行截断或分割。结合第 13 章学到的文本分割技术（RecursiveCharacterTextSplitter），将长文本切分成合适大小的片段再分别向量化。

❌ 错误 4：Ollama 连接失败

症状：FetchError: request to http://localhost:11434/api/embed failed

原因：Ollama 服务未启动或模型未下载。

解决：先运行 ollama serve 启动服务，再用 ollama pull nomic-embed-text 下载模型。确认 baseUrl 地址正确（默认 http://localhost:11434）。
