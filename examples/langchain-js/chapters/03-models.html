🧠 理解大语言模型

什么是大语言模型（LLM）

大语言模型（Large Language Model，简称 LLM）是现代人工智能领域最具突破性的技术之一。简单来说，LLM 就像一个超级博学的翻译官，你给它输入文字，它能理解你的意思并生成回复。它通过在海量文本数据上进行训练，学会了人类语言的模式、知识和推理能力。当你向它提出问题时，它会根据上下文预测最可能的回复内容，就像一个极其聪明的对话伙伴。

从技术角度看，LLM 的核心是一种称为 Transformer 的神经网络架构。它通过注意力机制（Attention Mechanism）来理解输入文本中每个词语之间的关系，并逐个生成输出 token。这个过程并不是简单的"查找答案"，而是一种概率生成——模型在每一步都计算下一个最可能出现的词语，然后将其拼接成完整的回复。正因如此，LLM 的输出具有一定的随机性，这也是为什么同一个问题可能会得到不同回答的原因。

Chat Models vs Completion Models

在 LangChain.js 的世界里，我们需要区分两种主要的模型类型。Chat Models（聊天模型）是目前的主流选择，它们设计用于处理结构化的消息对话。你可以发送系统消息（System Message）来设定角色，用户消息（User Message）来提问，以及助手消息（Assistant Message）来提供上下文。这种结构化的对话方式让模型能够更好地理解对话的语境和意图。

Completion Models（补全模型）是较早期的接口形式，它们接受一段纯文本作为输入并生成续写文本。这种方式没有消息角色的概念，所有内容都是一个连续的文本字符串。在现代 LangChain.js 开发中，我们强烈推荐使用 Chat Models，因为它们提供了更丰富的交互方式和更好的对话质量。本教程的所有示例都将基于 Chat Models。

Token 的概念与定价

Token 是 LLM 处理文本的基本单位。一个 token 大约等于一个英文单词的 3/4，或者一个中文字通常占用 1 到 2 个 token。理解 token 非常重要，因为它直接关系到 API 调用的成本。模型提供商通常按照输入 token 数量和输出 token 数量分别计费。例如，GPT-4o 的输入价格约为每百万 token 2.5 美元，输出价格约为每百万 token 10 美元。在实际开发中，控制 token 用量是优化成本的关键——你可以通过设置 maxTokens 参数来限制模型生成的最大 token 数量，避免意外的高额账单。

关键参数：Temperature 和 MaxTokens

Temperature（温度）是控制模型输出随机性的核心参数，取值范围通常为 0 到 2。当 temperature 设为 0 时，模型会始终选择概率最高的下一个 token，输出结果高度确定和可预测，非常适合需要精确回答的场景如代码生成、数据提取。当 temperature 设为较高值（如 0.7 或 1.0）时，模型会更倾向于选择概率稍低但更多样化的 token，输出更具创造性和多样性，适合创意写作、头脑风暴等场景。

MaxTokens（最大 token 数）则直接限制了模型单次回复的最大长度。设置合理的 maxTokens 不仅能控制成本，还能防止模型生成过于冗长的回复。例如，如果你只需要一个简短的摘要，可以将 maxTokens 设为 200；如果需要详细的分析报告，则可以设为 2000 或更高。需要注意的是，maxTokens 限制的是输出 token 数，不包含输入 token。

💡 Token 小贴士
不同模型对 token 的处理方式略有不同。OpenAI 使用 tiktoken 分词器，Anthropic 使用自己的分词器。在实际开发中，你可以使用 @anthropic-ai/tokenizer 或 js-tiktoken 等库来精确计算 token 数量。

🔧 initChatModel 详解

在 LangChain.js 中，创建模型实例的推荐方式是使用 initChatModel 函数。这是一个统一的工厂函数，无论你要连接 OpenAI、Anthropic 还是 Google 的模型，都只需要这一个入口。它会自动处理不同提供商之间的差异，让你用一致的接口来管理各种模型。这种方式的最大优势在于代码的可移植性——当你需要从一个模型切换到另一个模型时，通常只需要修改模型名称字符串，而不需要重构整个代码。

基本语法

TypeScript 
// 从 langchain 包中导入 initChatModel
import { initChatModel } from "langchain";

// 基本语法：initChatModel(modelName, options)
const model = await initChatModel("openai:gpt-4o", {
  temperature: 0.7,     // 控制输出随机性（0-2）
  maxTokens: 1000,      // 最大输出 token 数
  timeout: 30000,       // 请求超时时间（毫秒）
  maxRetries: 2,        // 最大重试次数
});

参数详解

参数名 类型 默认值 说明
modelName string 必填 模型名称，格式为 "提供商:模型名"，如 "openai:gpt-4o"
temperature number 1 控制输出随机性。0 = 确定性输出，2 = 最大随机性
maxTokens number 无限制 限制模型单次生成的最大 token 数量
timeout number 60000 单次 API 请求的超时时间（毫秒）
maxRetries number 2 请求失败后的最大重试次数

创建不同提供商的模型

initChatModel 的一大优势是使用统一的前缀语法来区分不同的模型提供商。你只需要在模型名称前加上提供商的标识符和冒号，LangChain 就会自动使用对应的客户端来连接。下面我们来看几个具体的例子。

创建 OpenAI 模型

TypeScript 
import { initChatModel } from "langchain";

// 创建 OpenAI GPT-4o 模型实例
// 需要设置环境变量 OPENAI_API_KEY
const openaiModel = await initChatModel("openai:gpt-4o", {
  temperature: 0,       // 使用确定性输出，适合精确任务
  maxTokens: 2000,      // 限制最多输出 2000 个 token
});

// 使用模型进行调用
const response = await openaiModel.invoke("你好，请用一句话介绍你自己。");
console.log(response.content);
你好！我是一个由 OpenAI 训练的大型语言模型，可以帮助你解答问题、创作内容和进行对话。

创建 Anthropic 模型

TypeScript 
import { initChatModel } from "langchain";

// 创建 Anthropic Claude 模型实例

  temperature: 0.5,     // 适度的创造性
  maxTokens: 1500,      // 限制输出长度
  timeout: 60000,       // 较长的超时时间，Claude 有时回复较慢
});

const response = await anthropicModel.invoke("请用三句话解释量子计算。");
console.log(response.content);
量子计算利用量子力学的基本原理（如叠加和纠缠）来处理信息。与传统计算机使用二进制比特不同，量子计算机使用量子比特，能够同时表示0和1的状态。这使得量子计算机在解决某些特定问题（如密码破解和分子模拟）时，理论上拥有远超传统计算机的能力。

🌐 多提供商支持

LangChain.js 的一大核心优势是它对多种模型提供商的统一支持。这意味着你可以在不同的大语言模型之间自由切换，而不需要大幅修改业务代码。这种灵活性在实际项目中非常有价值——你可以根据成本、性能、延迟等需求，为不同的使用场景选择最合适的模型。以下是 LangChain.js 当前支持的主要模型提供商及其特点。

提供商 代表模型 前缀标识 npm 包 环境变量
OpenAI gpt-4o、gpt-5.4 openai: @langchain/openai OPENAI_API_KEY
Anthropic claude-sonnet-4-6 anthropic: @langchain/anthropic ANTHROPIC_API_KEY
Google gemini-2.5-flash、gemini-2.5-pro google: @langchain/google-genai GOOGLE_API_KEY
Ollama llama3、mistral、qwen2.5 ollama: @langchain/ollama 无需 API Key

各提供商安装与初始化

要使用某个提供商的模型，你首先需要安装对应的 npm 包。虽然 initChatModel 是统一的入口，但它底层依赖各个提供商的专用包来处理实际的 API 通信。下面是各提供商的完整安装和初始化流程。

Bash

⚠️ API Key 安全提示
切勿将 API Key 直接硬编码在代码中。推荐使用 .env 文件配合 dotenv 包来管理密钥，并将 .env 文件添加到 .gitignore 中避免泄露。在生产环境中，应使用环境变量或密钥管理服务（如 AWS Secrets Manager、Vault）来存储敏感信息。

📡 模型调用方式

LangChain.js 为模型提供了三种核心调用方式，分别适用于不同的使用场景。理解这三种方式的区别和适用场景，是高效使用 LangChain 的基础。每种方式都返回标准的响应对象，你可以用一致的方式来处理结果。

1. invoke() — 单次调用

invoke() 是最基础也最常用的调用方式。它发送一个请求到模型并等待完整的响应返回。这种方式适用于不需要流式输出的场景，比如后端处理、批量分析等。调用时，你可以传入一个字符串或一组消息对象。返回值是一个 AIMessage 对象，其中 content 属性包含模型生成的文本内容。

TypeScript 
import { initChatModel } from "langchain";

// 初始化模型
const model = await initChatModel("openai:gpt-4o", {
  temperature: 0.7,
  maxTokens: 500,
});

// 方式一：直接传入字符串
const response1 = await model.invoke("什么是 TypeScript？");
console.log("回复内容：", response1.content);

// 方式二：传入消息数组（支持多轮对话上下文）
import { HumanMessage, SystemMessage } from "@langchain/core/messages";

const response2 = await model.invoke([
  new SystemMessage("你是一个资深的 TypeScript 专家，回答要简洁专业。"),
  new HumanMessage("TypeScript 和 JavaScript 的主要区别是什么？"),
]);
console.log("专业回复：", response2.content);

// 查看响应的元数据
console.log("使用的 token 数：", response2.usage_metadata);
回复内容：TypeScript 是 JavaScript 的超集，添加了静态类型系统，可以在编译时发现错误，提高代码的可维护性。 专业回复：核心区别在于类型系统。TypeScript 引入了静态类型检查、接口、泛型等特性，编译后生成纯 JavaScript 代码。 使用的 token 数：{ input_tokens: 45, output_tokens: 62, total_tokens: 107 }

2. stream() — 流式响应

stream() 方法允许你以流的方式接收模型的响应。模型每生成一小段内容就会立即推送给你，而不需要等待整个回复完成。这种方式在用户体验方面有巨大优势——用户可以在模型还在"思考"的时候就开始看到部分内容，大幅减少感知等待时间。几乎所有面向终端用户的 AI 应用都应该使用流式响应。返回的是一个异步可迭代对象（AsyncIterable），你可以用 for await...of 循环来逐块接收数据。

TypeScript 
import { initChatModel } from "langchain";

const model = await initChatModel("openai:gpt-4o", {
  temperature: 0.7,
});

// 使用 stream() 获取流式响应
const stream = await model.stream("请用五句话介绍 Node.js 的核心特性。");

// 逐块接收并输出
let fullResponse = "";
for await (const chunk of stream) {
  // 每个 chunk 都包含一小段文本
  const content = chunk.content;
  if (typeof content === "string") {
    process.stdout.write(content);  // 实时输出，不换行
    fullResponse += content;
  }
}

console.log("\n--- 流式输出结束 ---");
console.log("总字符数：", fullResponse.length);
Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时环境。它采用事件驱动的非阻塞 I/O 模型，使其轻量且高效。Node.js 拥有世界上最大的开源包管理生态系统 npm。它特别适合构建高并发的网络应用和实时服务。Node.js 的单线程事件循环模型简化了并发编程的复杂度。 --- 流式输出结束 --- 总字符数：168

3. batch() — 批量调用

batch() 方法允许你同时发送多个请求到模型。它内部使用并发控制来优化请求的执行效率，比逐个调用 invoke() 要高效得多。当你需要对多条数据进行相同的 AI 处理时（比如批量分类、批量翻译），batch() 是最佳选择。每个请求仍然是独立的，它们之间不会共享上下文。

TypeScript 
import { initChatModel } from "langchain";

const model = await initChatModel("openai:gpt-4o", {
  temperature: 0,       // 使用确定性输出，保证分类一致性
  maxTokens: 20,        // 限制输出长度，只需分类标签
});

// 准备多个待分析的文本
const texts = [
  "今天天气真好，心情很愉快！",
  "这个产品质量太差了，非常失望。",
  "请问你们的营业时间是几点到几点？",
  "刚收到快递，包装很精美，非常满意！",
];

// 构造批量请求 — 每条文本要求模型判断情感
const prompts = texts.map(
  (text) => `请判断以下文本的情感（正面/负面/中性），只回复情感标签：\n"${text}"`
);

// 使用 batch() 并发调用
const results = await model.batch(prompts);

// 输出分类结果
results.forEach((result, index) => {
  console.log(`文本 ${index + 1}: "${texts[index].slice(0, 20)}..." → ${result.content}`);
});
文本 1: "今天天气真好，心情很愉快！..." → 正面 文本 2: "这个产品质量太差了，非常失..." → 负面 文本 3: "请问你们的营业时间是几点到..." → 中性 文本 4: "刚收到快递，包装很精美，非..." → 正面
💡 调用方式选择指南
  • invoke() — 最常用，适合单次请求，后端逻辑处理
  • stream() — 面向用户的应用首选，提升体验感
  • batch() — 批量数据处理，提高并发效率

🛡️ 连接弹性与重试

在生产环境中调用大语言模型时，网络不稳定、API 速率限制、服务器临时故障等问题都是不可避免的。LangChain.js 内置了强大的连接弹性机制，能够自动处理这些异常情况，确保你的应用在面对临时故障时依然能够稳定运行。这些机制是默认开启的，你只需要通过简单的配置就可以定制其行为。

自动指数退避重试

当 API 请求失败时，LangChain.js 不会立即重试，而是采用指数退避（Exponential Backoff）策略。这意味着每次重试之间的等待时间会按指数级增长——第一次失败后等待 1 秒，第二次等待 2 秒，第三次等待 4 秒，依此类推。这种策略的目的是避免在服务器压力大时发送大量重试请求，从而加剧问题。指数退避是处理 API 速率限制的最佳实践，被所有主流云服务商推荐使用。

TypeScript 
import { initChatModel } from "langchain";

// 配置重试策略
const resilientModel = await initChatModel("openai:gpt-4o", {
  temperature: 0.7,
  maxRetries: 3,        // 最多重试 3 次
  timeout: 15000,       // 每次请求 15 秒超时
});

try {
  // 即使遇到临时故障，LangChain 会自动重试
  const response = await resilientModel.invoke("你好！");
  console.log("成功：", response.content);
} catch (error) {
  // 只有在所有重试都失败后，才会抛出错误
  console.error("所有重试均失败：", error.message);
}

处理速率限制（429 错误）

速率限制是调用 API 时最常见的问题之一。当你的请求频率超过模型提供商的限制时，API 会返回 HTTP 429 状态码。LangChain.js 能够自动识别 429 响应，并根据响应头中的 Retry-After 字段来决定等待时间。如果你使用的是免费套餐或低级别计划，建议在代码中显式增加 maxRetries 的值，以获得更好的容错能力。

TypeScript 
import { initChatModel } from "langchain";

// 对于频繁调用的场景，增加重试次数和超时时间
const model = await initChatModel("anthropic:claude-sonnet-4-6", {
  temperature: 0.5,
  maxRetries: 5,        // 增加重试次数以应对速率限制
  timeout: 60000,       // 增加超时时间
});

// 封装一个带错误处理的调用函数
async function safeInvoke(prompt: string): Promise {
  try {
    const response = await model.invoke(prompt);
    return response.content as string;
  } catch (error: any) {
    if (error.status === 429) {
      console.warn("⚠️ 触发速率限制，请稍后再试");
      // 可以在此处实现自定义的等待逻辑
    } else if (error.status >= 500) {
      console.warn("⚠️ 服务端错误，已耗尽重试次数");
    }
    throw error;
  }
}

处理服务端错误（5xx）

服务端错误（HTTP 500、502、503 等）通常表示模型提供商的服务出现了临时问题。这类错误通常是暂时性的，稍后重试就可能成功。LangChain.js 的重试机制会自动处理这些错误。在极端情况下，如果你的应用对可用性要求非常高，可以考虑实现多模型降级策略——当主模型不可用时，自动切换到备用模型。

TypeScript 
import { initChatModel } from "langchain";

// 多模型降级策略示例
async function invokeWithFallback(prompt: string): Promise {
  // 按优先级排列的模型列表
  const models = [
    { name: "openai:gpt-4o", provider: "OpenAI" },
    { name: "anthropic:claude-sonnet-4-6", provider: "Anthropic" },
    { name: "google:gemini-2.5-flash", provider: "Google" },
  ];

  for (const { name, provider } of models) {
    try {
      const model = await initChatModel(name, {
        temperature: 0.7,
        maxRetries: 1,      // 每个模型只试一次，快速降级
        timeout: 10000,
      });
      const response = await model.invoke(prompt);
      console.log(`✅ 使用 ${provider} 成功响应`);
      return response.content as string;
    } catch (error) {
      console.warn(`❌ ${provider} 失败，尝试下一个模型...`);
    }
  }

  throw new Error("所有模型均不可用");
}

🔄 实战：构建多模型切换器

在实际开发中，你经常需要对比不同模型对同一个问题的回答质量，或者根据不同的需求动态切换模型。下面我们来构建一个实用的多模型切换器，它支持在 OpenAI 和 Anthropic 之间自由切换，并且可以方便地对比两个模型的回复质量。这个例子也展示了如何在实际项目中优雅地管理多个模型实例。

TypeScript 
/**
 * 多模型切换器 - 支持 OpenAI 和 Anthropic 模型的切换与对比
 * 文件名：multi-model-switcher.ts
 */
import { initChatModel } from "langchain";
import type { BaseChatModel } from "@langchain/core/language_models/chat_models";

// 定义支持的模型提供商
type ModelProvider = "openai" | "anthropic";

// 模型配置映射表
const MODEL_CONFIGS: Record = {
  openai: {
    modelName: "openai:gpt-4o",
    displayName: "OpenAI GPT-4o",
  },
  anthropic: {
    modelName: "anthropic:claude-sonnet-4-6",
    displayName: "Anthropic Claude Sonnet 4",
  },
};

// 模型缓存 — 避免重复创建实例
const modelCache = new Map();

/**
 * 获取或创建模型实例
 * 使用缓存机制避免重复初始化
 */
async function getModel(
  provider: ModelProvider,
  temperature: number = 0.7
): Promise {
  const cacheKey = `${provider}-${temperature}`;

  if (modelCache.has(cacheKey)) {
    return modelCache.get(cacheKey)!;
  }

  const config = MODEL_CONFIGS[provider];
  console.log(`🔧 正在初始化 ${config.displayName}...`);

  const model = await initChatModel(config.modelName, {
    temperature,
    maxTokens: 1000,
    maxRetries: 2,
  });

  modelCache.set(cacheKey, model);
  return model;
}

/**
 * 使用指定模型回答问题
 */
async function ask(
  provider: ModelProvider,
  question: string,
  temperature: number = 0.7
): Promise<{ answer: string; duration: number }> {
  const model = await getModel(provider, temperature);
  const config = MODEL_CONFIGS[provider];

  const startTime = Date.now();
  console.log(`\n📤 发送问题到 ${config.displayName}...`);

  const response = await model.invoke(question);
  const duration = Date.now() - startTime;

  return {
    answer: response.content as string,
    duration,
  };
}

/**
 * 对比两个模型的回答
 * 同时向两个模型发送相同的问题，并展示对比结果
 */
async function compare(
  question: string,
  temperature: number = 0.5
): Promise {
  console.log("═".repeat(60));
  console.log(`📝 问题：${question}`);
  console.log("═".repeat(60));

  // 并行调用两个模型
  const [openaiResult, anthropicResult] = await Promise.all([
    ask("openai", question, temperature),
    ask("anthropic", question, temperature),
  ]);

  // 输出对比结果
  console.log("\n" + "─".repeat(60));
  console.log(`🤖 ${MODEL_CONFIGS.openai.displayName}（${openaiResult.duration}ms）：`);
  console.log("─".repeat(60));
  console.log(openaiResult.answer);

  console.log("\n" + "─".repeat(60));
  console.log(`🧠 ${MODEL_CONFIGS.anthropic.displayName}（${anthropicResult.duration}ms）：`);
  console.log("─".repeat(60));
  console.log(anthropicResult.answer);
  console.log();
}

// ====== 使用示例 ======
async function main() {
  // 示例 1：对比创意写作能力
  await compare("用一个比喻来形容编程的乐趣，限 50 字以内。");

  // 示例 2：对比知识问答能力
  await compare("TCP 和 UDP 协议的核心区别是什么？");

  // 示例 3：切换到单个模型使用
  const result = await ask("openai", "生成五个适合 TypeScript 项目的 npm 包名推荐。");
  console.log(`\n🤖 OpenAI 的推荐（${result.duration}ms）：`);
  console.log(result.answer);
}

main().catch(console.error);

🏠 本地模型：使用 Ollama

不是所有场景都需要调用云端 API。在某些情况下——比如处理敏感数据、离线环境、或者想零成本体验大语言模型——使用本地模型是一个非常好的选择。Ollama 是一个流行的本地大模型运行工具，它让你能够在自己的电脑上运行 Llama 3、Mistral、Qwen 等开源模型，无需任何 API Key，也不会产生任何费用。LangChain.js 通过 @langchain/ollama 包提供了对 Ollama 的无缝集成。

步骤一：安装 Ollama

首先，你需要在电脑上安装 Ollama。访问 ollama.com 下载对应操作系统的安装包。安装完成后，Ollama 会自动在后台启动一个本地服务，默认监听 http://localhost:11434。

Bash 
# macOS / Linux 安装（一键脚本）
curl -fsSL https://ollama.com/install.sh | sh

# Windows 用户请从官网下载安装包
# https://ollama.com/download

# 验证安装是否成功
ollama --version

步骤二：拉取模型

安装好 Ollama 后，你需要下载一个模型到本地。Ollama 支持多种开源模型，你可以根据自己的硬件配置选择合适的模型。一般来说，8B 参数的模型需要至少 8GB 内存，70B 参数的模型需要至少 40GB 内存。以下是几个推荐的模型。

📝 自测题 — 第 3 章
问题 1：initChatModel 函数中，模型名称的格式是什么？请给出一个连接 Anthropic Claude 模型的示例。

模型名称的格式是 "提供商前缀:模型名"。连接 Anthropic Claude 的示例：

const model = await initChatModel("anthropic:claude-sonnet-4-6", {
  temperature: 0.7,
});

提供商前缀包括 openai:、anthropic:、google:、ollama: 等。

问题 2：temperature 参数设为 0 和设为 1 时，模型的行为有何不同？分别适合什么场景？

temperature = 0：模型会始终选择概率最高的 token，输出高度确定和可重复。适合代码生成、数据提取、分类任务等需要精确回答的场景。

temperature = 1：模型有更高的概率选择概率较低的 token，输出更具多样性和创造性。适合创意写作、头脑风暴、故事创作等场景。

问题 3：invoke()、stream() 和 batch() 三种调用方式各自的特点和适用场景是什么？

invoke()：单次同步调用，等待完整响应返回。适合后端逻辑处理、不需要实时输出的场景。

stream()：流式调用，逐块接收响应。适合面向用户的应用，可以实时展示输出，提升用户体验。

batch()：批量并发调用，同时处理多个请求。适合批量数据分析、批量分类等需要处理多条数据的场景。

问题 4：当 API 返回 429 错误时，LangChain.js 会如何处理？什么是指数退避策略？

当 API 返回 429（速率限制）错误时，LangChain.js 会自动进行重试。它使用指数退避（Exponential Backoff）策略，即每次重试之间的等待时间按指数级增长（如 1s、2s、4s、8s...），以避免给服务器施加更大压力。最大重试次数由 maxRetries 参数控制，默认为 2 次。只有在所有重试都失败后，才会向调用方抛出错误。

🐛 常见错误

❌ 错误 1：API Key 未设置

错误信息：OpenAI API key not found. Please set the OPENAI_API_KEY environment variable.

原因：没有设置对应的 API Key 环境变量。

解决方案：

  • 在终端中设置环境变量：export OPENAI_API_KEY="sk-your-key"
  • 或者使用 .env 文件配合 dotenv 包：import "dotenv/config";
  • 确保环境变量名称正确：OPENAI_API_KEY、ANTHROPIC_API_KEY、GOOGLE_API_KEY
❌ 错误 2：模型名称不存在

错误信息：The model 'gpt-5-turbo' does not exist or you do not have access to it.

原因：模型名称拼写错误，或者模型已更名/下线。

解决方案：

  • 检查模型名称拼写是否正确（区分大小写）
  • 确认你的 API 账户有权访问该模型
  • 查阅模型提供商的官方文档，确认最新的可用模型列表
  • 常见正确名称：gpt-4o（非 gpt-4o-turbo）、claude-sonnet-4-6（非 claude-3-sonnet）
❌ 错误 3：速率限制被触发

错误信息：Rate limit reached for requests. Please retry after 30 seconds.

原因：API 请求频率超过账户的速率限制。

解决方案：

  • 增加 maxRetries 参数值（如 maxRetries: 5）让 LangChain 自动重试
  • 使用 batch() 时控制并发数量，避免同时发送过多请求
  • 在应用层实现请求队列，控制整体请求速率
  • 升级 API 订阅计划以获得更高的速率限制
❌ 错误 4：请求超时

错误信息：Request timed out after 60000ms

原因：模型响应时间超过了设定的 timeout 值。这在大模型生成长文本时更容易发生。

解决方案：

  • 增加 timeout 参数值：timeout: 120000（120 秒）
  • 使用 stream() 代替 invoke()，流式调用的超时管理更加灵活
  • 减少 maxTokens 以缩短生成时间
  • 检查网络连接是否稳定，特别是访问海外 API 时

在本章中，我们系统学习了 LangChain.js 中模型的核心知识：

  • 理解 LLM — 大语言模型是基于 Transformer 架构的概率生成模型，通过 token 逐个生成回复。Chat Models 是当前推荐的模型类型，通过 temperature 和 maxTokens 控制输出特性

  • 连接弹性 — 内置指数退避重试机制，自动处理 429 速率限制和 5xx 服务端错误，支持自定义重试策略
