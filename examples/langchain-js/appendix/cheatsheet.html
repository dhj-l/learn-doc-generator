📦 安装与导入

LangChain 采用模块化包结构，按需安装所需的功能包。核心包和各集成包相互独立。

包名 / 操作 说明 快速示例
@langchain/core 核心抽象层：消息、工具、输出解析器等基础类型 npm install @langchain/core
@langchain/langgraph Agent 运行时、状态图、检查点、Store 等 npm install @langchain/langgraph
@langchain/openai OpenAI / Azure OpenAI 集成 npm install @langchain/openai
@langchain/anthropic Anthropic Claude 集成 npm install @langchain/anthropic
@langchain/google-genai Google Gemini 集成 npm install @langchain/google-genai
initChatModel 导入 统一模型初始化工厂，支持多 provider import { initChatModel } from "@langchain/langgraph"
createAgent 导入 创建 Agent 的核心函数 import { createAgent } from "@langchain/langgraph"
消息类型导入 各类消息构造函数 import { SystemMessage, HumanMessage, AIMessage, ToolMessage } from "@langchain/core/messages"
工具定义导入 定义工具的 tool() 函数与 z 校验 import { tool } from "@langchain/core/tools"

🤖 Agent

createAgent 是 LangChain v1.x 创建 Agent 的标准方式。Agent = Model + Harness，负责管理模型调用、工具选择和对话流程。

createAgent 参数

参数 说明 快速示例
model 语言模型实例或模型标识字符串 model: "openai:gpt-4o"
tools Agent 可调用的工具数组 tools: [searchTool, calculatorTool]
systemPrompt 系统提示词，定义 Agent 的角色和行为 systemPrompt: "你是一个天气助手"
checkpointer 检查点存储，用于持久化会话状态 checkpointer: new MemorySaver()
middleware 中间件数组，在模型调用前后执行逻辑 middleware: [summarizationMiddleware()]
responseFormat 结构化输出格式定义 responseFormat: z.object({ answer: z.string() })

Agent 方法

方法 说明 快速示例
agent.invoke() 同步调用 Agent，等待完整响应后返回 const result = await agent.invoke({ messages: [new HumanMessage("你好")] })
agent.stream() 流式调用 Agent，逐步返回事件流 for await (const event of agent.stream(input, { streamMode: "updates" })) { ... }

快速创建示例

TypeScript
import { createAgent } from "@langchain/langgraph";
import { MemorySaver } from "@langchain/langgraph";

const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [myTool],
  systemPrompt: "你是一个有用的助手",
  checkpointer: new MemorySaver(),
});

// 调用
const result = await agent.invoke(
  { messages: [{ role: "user", content: "你好" }] },
  { configurable: { thread_id: "session-1" } }
);

🧠 模型 Models

LangChain 使用 initChatModel 统一初始化不同提供商的模型，也支持直接实例化具体模型类。

初始化与调用

API / 函数 说明 快速示例
initChatModel() 统一工厂函数，通过字符串标识切换 provider const model = await initChatModel("openai:gpt-4o")
model.invoke() 同步调用，发送消息数组并返回完整 AI 消息 const response = await model.invoke([new HumanMessage("你好")])
model.stream() 流式调用，返回 AsyncIterable 逐步生成 token for await (const chunk of model.stream(messages)) { process.stdout.write(chunk.content) }
model.batch() 批量调用，一次发送多组消息并行处理 const results = await model.batch([[msg1], [msg2], [msg3]])
model.withConfig() 创建带有额外配置的新模型实例（不修改原实例） const cold = model.withConfig({ temperature: 0 })
model.bindTools() 将工具绑定到模型，使其能够生成工具调用 const modelWithTools = model.bindTools([myTool])

常用模型参数

参数 说明 快速示例
temperature 输出随机性，0 = 确定性，1 = 更随机 temperature: 0.7
maxTokens 最大输出 token 数 maxTokens: 4096
timeout 请求超时时间（毫秒） timeout: 30000
maxRetries 请求失败后最大重试次数 maxRetries: 3
apiKey API 密钥（通常用环境变量，此为覆盖方式） apiKey: "sk-..."
model 具体模型名称 model: "gpt-4o"

💬 消息 Messages

消息是 Agent 与模型交互的基本单位。每条消息都有 role（角色）和 content（内容），不同类型的消息在对话中扮演不同角色。

消息类型 说明 快速示例
SystemMessage 系统消息，定义 AI 的角色和行为规则，通常放在对话开头 new SystemMessage("你是一个翻译助手，只翻译不解释")
HumanMessage 用户消息，代表用户发送的输入内容 new HumanMessage("今天天气怎么样？")
AIMessage AI 消息，代表模型生成的回复，可能包含工具调用 new AIMessage("今天天气晴朗，温度 25 度。")
ToolMessage 工具消息，包含工具执行的结果，需指定 tool_call_id new ToolMessage({ content: "晴天 25°C", tool_call_id: "call_123" })
message.content 消息的文本内容，可以是字符串或结构化内容块 const text = response.content // string | ContentBlock[]
message.tool_calls AI 消息中的工具调用请求数组（AIMessage 特有） aiMsg.tool_calls // [{ name: "search", args: { q: "..." }, id: "..." }]
role 属性 消息角色标识符 message.role // "system" | "human" | "ai" | "tool"

快捷构造（对象形式）

除了使用类构造函数外，也可以直接用对象字面量构建消息数组：

TypeScript
const messages = [
  { role: "system", content: "你是一个助手" },
  { role: "user", content: "你好" },
  { role: "assistant", content: "你好！有什么我能帮你的？" },
  { role: "user", content: "今天天气如何？" },
];

🔧 工具 Tools

工具让 Agent 能够执行具体操作。通过 tool() 函数定义工具，使用 Zod Schema 描述参数，Agent 会在需要时自动选择和调用合适的工具。

tool() 函数

API / 参数 说明 快速示例
tool(fn, options) 定义一个工具，第一个参数是执行函数，第二个是配置 const myTool = tool(async (input) => ..., { name: "...", schema: z.object({...}) })
name 工具名称，模型通过此名称引用工具 name: "search_web"
description 工具功能描述，帮助模型决定何时使用此工具 description: "搜索互联网获取最新信息"
schema Zod Schema，定义工具接受的输入参数及类型 schema: z.object({ query: z.string().describe("搜索关键词") })
returnDirect 设为 true 时，工具结果直接返回给用户，不经模型总结 returnDirect: true

runtime 上下文

工具函数的第二个参数 runtime 提供了运行时上下文访问能力：

属性 / 方法 说明 快速示例
runtime.context 外部注入的上下文数据（如用户信息、配置等） const userId = runtime.context.userId
runtime.store 持久化 KV 存储，跨会话保存数据 await runtime.store.put(["user", id], "prefs", data)
runtime.writer 流式写入器，工具执行过程中可实时推送中间结果 runtime.writer({ status: "正在搜索..." })

完整工具定义示例

TypeScript
import { tool } from "@langchain/core/tools";
import { z } from "zod";

const weatherTool = tool(
  async ({ city }, runtime) => {
    // 可通过 runtime 访问上下文和存储
    const apiKey = runtime.context.weatherApiKey;
    const response = await fetch(`/api/weather?city=${city}&key=${apiKey}`);
    const data = await response.json();
    return `${city}：${data.condition}，温度 ${data.temp}°C`;
  },
  {
    name: "get_weather",
    description: "查询指定城市的当前天气",
    schema: z.object({
      city: z.string().describe("城市名称，如 北京、上海"),
    }),
    returnDirect: false,
  }
);

📡 流式传输 Streaming

流式传输允许逐步接收 Agent 的输出，避免长时间等待。LangChain 支持多种 streamMode 和事件类型，适用于不同场景。

streamMode 选项

streamMode 值 说明 快速示例
"updates" 节点级别的完整更新（默认），每个节点完成时发出 agent.stream(input, { streamMode: "updates" })
"messages" Token 级流式输出，逐 token 推送 AI 回复 agent.stream(input, { streamMode: "messages" })
"events" 最细粒度事件流，包含所有内部事件 agent.stream(input, { streamMode: "events" })
"values" 每次更新返回完整的状态快照 agent.stream(input, { streamMode: "values" })
多模式组合 可同时使用多种模式 agent.stream(input, { streamMode: ["updates", "messages"] })

事件迭代模式

模式 说明 快速示例
for await 基础迭代 使用 for await...of 遍历事件流 for await (const event of stream) { console.log(event) }
Token 输出 messages 模式下逐 token 输出到终端 for await (const [msg, metadata] of stream) { if (msg.content) process.stdout.write(String(msg.content)) }
按节点过滤 在 updates 模式下按节点名过滤事件 for await (const event of stream) { if (event.agent) { /* agent 节点事件 */ } }
中断与消费 可随时 break 中断流消费 for await (const event of stream) { if (done) break; }

📐 结构化输出

结构化输出让模型返回符合预定义 Schema 的 JSON 数据，而非自由文本。LangChain 提供两种策略：基于工具调用和基于 provider 原生能力。

策略选择

策略 说明 快速示例
toolStrategy() 利用工具调用机制实现结构化输出，兼容所有模型（默认策略） responseFormat: toolStrategy(mySchema)
providerStrategy() 使用 provider 原生结构化输出能力（如 OpenAI response_format），性能更优 responseFormat: providerStrategy(mySchema)
responseFormat 参数 在 createAgent 中直接传入 Zod Schema，自动选择最佳策略 responseFormat: z.object({ sentiment: z.enum(["pos", "neg"]) })

Zod Schema 基础

Zod 类型 说明 快速示例
z.string() 字符串类型 z.string().describe("用户姓名")
z.number() 数值类型 z.number().min(0).max(100)
z.boolean() 布尔类型 z.boolean().default(false)
z.enum() 枚举类型，限定可选值 z.enum(["positive", "negative", "neutral"])
z.array() 数组类型 z.array(z.string())
z.object() 对象类型，组合多个字段 z.object({ name: z.string(), age: z.number() })
.describe() 为字段添加描述，帮助模型理解预期内容 z.string().describe("一段简短的情感分析总结")
.optional() 标记字段为可选 z.string().optional()

完整示例

TypeScript
import { z } from "zod";
import { createAgent } from "@langchain/langgraph";

const AnalysisSchema = z.object({
  sentiment: z.enum(["positive", "negative", "neutral"]).describe("情感倾向"),
  confidence: z.number().min(0).max(1).describe("置信度"),
  summary: z.string().describe("一句话总结"),
  keywords: z.array(z.string()).describe("关键词列表"),
});

const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [],
  responseFormat: AnalysisSchema,
});

const result = await agent.invoke({
  messages: [{ role: "user", content: "分析这段文本的情感..." }],
});
// result.structuredResponse 符合 AnalysisSchema 类型

🔗 中间件 Middleware

中间件在模型调用前后插入自定义逻辑，常用于消息压缩、人机协作、日志记录等场景。多个中间件按数组顺序依次执行。

内置中间件

中间件 说明 快速示例
summarizationMiddleware 当消息过长时自动压缩历史消息为摘要，避免超出上下文窗口 summarizationMiddleware({ maxTokens: 4000 })
humanInTheLoopMiddleware 在工具执行前暂停，等待人工审批后继续（需配合 checkpointer） humanInTheLoopMiddleware({ tools: ["sensitive_tool"] })

自定义中间件模式

模式 说明 快速示例
beforeModel 钩子 在模型调用前执行，可修改消息列表 beforeModel: async (state) => { logMessages(state.messages); return state; }
afterModel 钩子 在模型响应后执行，可修改或过滤 AI 回复 afterModel: async (state) => { validateResponse(state.messages); return state; }
中间件注册 通过 createAgent 的 middleware 数组注册 createAgent({ model, tools, middleware: [myMiddleware] })

使用示例

TypeScript
import { createAgent } from "@langchain/langgraph";
import {
  summarizationMiddleware,
  humanInTheLoopMiddleware,
} from "@langchain/langgraph";

const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [sensitiveTool, safeTool],
  systemPrompt: "你是一个数据库助手",
  checkpointer: new MemorySaver(),
  middleware: [
    summarizationMiddleware({ maxTokens: 4000 }),
    humanInTheLoopMiddleware({
      tools: ["sensitiveTool"],  // 仅对敏感工具暂停审批
    }),
  ],
});

🧩 记忆系统

记忆系统让 Agent 能够跨多轮对话保持上下文。LangChain 通过 checkpointer 实现会话级别的状态持久化，通过 Store 实现跨会话的长期存储。

API / 概念 说明 快速示例
MemorySaver 内存检查点存储，适合开发调试，重启后数据丢失 import { MemorySaver } from "@langchain/langgraph"
checkpointer 参数 传入 createAgent 启用会话持久化 createAgent({ ..., checkpointer: new MemorySaver() })
thread_id 会话线程 ID，不同 ID 对应独立的对话上下文 { configurable: { thread_id: "user-123" } }
InMemoryStore 内存 KV 存储，用于跨会话持久化数据（如用户偏好） import { InMemoryStore } from "@langchain/langgraph"
store.put() 在 Store 中写入数据 await store.put(["users", userId], "preferences", { lang: "zh" })
store.get() 从 Store 中读取数据 const prefs = await store.get(["users", userId], "preferences")

会话记忆示例

TypeScript
import { createAgent } from "@langchain/langgraph";
import { MemorySaver } from "@langchain/langgraph";

// 创建带记忆的 Agent
const agent = createAgent({
  model: "openai:gpt-4o",
  tools: [myTools],
  checkpointer: new MemorySaver(),
});

// 第一轮对话
await agent.invoke(
  { messages: [{ role: "user", content: "我叫小明" }] },
  { configurable: { thread_id: "session-1" } }
);

// 第二轮对话 —— Agent 记得"小明"
const result = await agent.invoke(
  { messages: [{ role: "user", content: "我叫什么名字？" }] },
  { configurable: { thread_id: "session-1" } }  // 同一 thread_id
);

⚙️ 环境变量

LangChain 通过环境变量配置 API 密钥和调试选项。推荐使用 .env 文件配合 dotenv 包管理敏感信息。

环境变量 说明 快速示例
OPENAI_API_KEY OpenAI API 密钥，使用 GPT 系列模型时必需 OPENAI_API_KEY=sk-proj-xxxx
ANTHROPIC_API_KEY Anthropic API 密钥，使用 Claude 系列模型时必需 ANTHROPIC_API_KEY=sk-ant-xxxx
GOOGLE_API_KEY Google AI API 密钥，使用 Gemini 系列模型时必需 GOOGLE_API_KEY=AIzaSy-xxxx
LANGSMITH_TRACING 启用 LangSmith 调试追踪，设为 "true" 开启 LANGSMITH_TRACING=true
LANGSMITH_API_KEY LangSmith 平台 API 密钥，用于追踪和调试 LANGSMITH_API_KEY=lsv2_pt-xxxx

.env 文件配置

.env
# 模型 API 密钥（至少配置一个）
OPENAI_API_KEY=sk-proj-your-key-here
ANTHROPIC_API_KEY=sk-ant-your-key-here

# LangSmith 调试（可选但推荐）
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=lsv2_pt-your-key-here
TypeScript
// 在项目入口文件顶部加载 .env
import "dotenv/config";
// 之后 process.env.OPENAI_API_KEY 等即可使用
