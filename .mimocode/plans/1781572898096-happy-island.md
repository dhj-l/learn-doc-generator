# Learning Enhancer Skill 设计计划

## 概述

创建一个名为 `learning-enhancer` 的独立skill，用于将专业且枯燥的教学文档内容简化为简单易懂的内容，帮助用户由浅入深学习并掌握知识。对于专业概念级别的理论，保留原文并在下方添加简单易懂的解释。

## 用户需求确认

1. **Skill名称**：learning-enhancer
2. **输出格式**：修改原文档（覆盖原始文件）
3. **文档类型**：Markdown和纯文本文件
4. **专业概念处理**：保留原文+在下方添加简单易懂的解释
5. **与现有skill关系**：完全独立，不依赖learn-doc-generator

## 设计方案

### 1. Skill结构

```
learning-enhancer/
├── SKILL.md                    # 主要skill定义文件
├── agents/
│   └── openai.yaml             # UI元数据
└── references/
    └── concept-identifier.md   # 专业概念识别规则和解释策略
```

### 2. 触发条件

当用户提供文档并请求简化时触发。例如：
- "简化这个文档"
- "让这个文档更易懂"
- "为这个文档添加解释"
- "帮我理解这个技术文档"
- "把这个专业文档变得简单一些"
- "enhance this document"
- "simplify this doc"

### 3. 工作流程

#### 阶段1：文档分析
1. 读取用户提供的文档（Markdown或纯文本）
2. 分析文档结构（章节、标题、段落）
3. 识别专业概念和技术术语
4. 评估文档的复杂度和目标受众

#### 阶段2：内容简化
1. 保持原始文档结构不变
2. 对每个段落进行简化：
   - 用简单语言重写复杂句子
   - 添加类比和日常例子
   - 将长段落拆分为短段落
   - 添加过渡句，使内容更连贯
3. 对专业概念处理：
   - 保留原始专业术语和定义
   - 在专业概念下方添加简单解释
   - 使用类比、比喻或日常例子解释概念
   - 确保解释与原文内容一致

#### 阶段3：质量检查
1. 检查简化后的内容是否准确传达了原始含义
2. 确保专业概念的解释正确无误
3. 验证文档结构是否完整保持
4. 检查是否有遗漏的重要信息

#### 阶段4：文档更新
1. 将简化后的内容写回原始文件
2. 保持原始文件格式（Markdown语法、文本缩进等）
3. 确保所有链接、引用等保持不变

### 4. 专业概念识别策略

#### 识别规则
1. **技术术语**：编程语言、框架、库的特定术语
2. **学术概念**：数学、物理、计算机科学等领域的专业概念
3. **行业术语**：特定行业的专业用语
4. **复杂定义**：包含多个从句或抽象描述的定义

#### 解释策略
1. **类比解释**：用日常事物比喻专业概念
2. **分步解释**：将复杂概念分解为简单步骤
3. **示例说明**：提供具体代码示例或实际应用场景
4. **对比说明**：与类似概念进行对比，突出特点

### 5. 输出格式

修改后的文档应保持：
- 原始Markdown语法（标题、列表、代码块等）
- 原始文件编码（UTF-8）
- 原始图片链接和引用
- 添加的解释使用统一的格式（如引用块或特定样式）

### 6. 质量保证

1. **准确性检查**：简化后的内容不改变原始技术含义
2. **完整性检查**：所有重要信息都被保留
3. **可读性检查**：简化后的内容确实更易理解
4. **一致性检查**：专业术语的解释在全文中保持一致

## 实现步骤

### 步骤1：确定skill存放位置
根据现有项目结构，skill需要同时存放在两个位置：
1. **根目录**：`learning-enhancer.md`（Claude Code命令格式）
2. **Reasonix目录**：`.reasonix/skills/learning-enhancer/SKILL.md`（Reasonix平台格式）

### 步骤2：创建skill目录结构
1. 在 `.reasonix/skills/` 目录下创建 `learning-enhancer/` 目录
2. 创建 `SKILL.md` 文件（主要内容）
3. 创建 `agents/openai.yaml` 文件（UI元数据）
4. 创建 `references/concept-identifier.md` 文件（专业概念识别规则）
5. 在根目录创建 `learning-enhancer.md` 文件（与`.reasonix/skills/`中的内容相同）

### 步骤3：编写SKILL.md
1. 定义YAML前置元数据（name和description）
2. 编写详细的skill指令和工作流程
3. 添加示例和最佳实践
4. 包含质量检查标准

### 步骤4：编写参考文件
1. 创建 `concept-identifier.md`，定义专业概念识别规则
2. 提供解释策略和示例
3. 包含常见专业领域的术语列表

### 步骤5：生成UI元数据
1. 使用 `generate_openai_yaml.py` 生成 `agents/openai.yaml`
2. 确保显示名称、简短描述和默认提示词准确反映skill功能

### 步骤6：验证skill
1. 运行 `quick_validate.py` 验证skill结构
2. 测试skill在实际文档上的效果
3. 根据测试结果进行迭代优化
